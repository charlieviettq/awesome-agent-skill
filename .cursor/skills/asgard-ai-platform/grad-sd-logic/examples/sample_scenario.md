# Example: 工業設備製造商轉型服務生態系

## Scenario

**公司**：台灣機械股份有限公司（TM）——台中精密工具機製造商，年營收 NT$28 億，主要出口歐洲與東南亞

**問題陳述**（產品經理提問）：

> 我們賣 CNC 工具機已經 40 年了，利潤越來越薄。競爭對手從中國、韓國殺進來，我們的設備功能已經很難差異化。業務部說要「提供解決方案」，但我們根本不知道客戶買機台之後怎麼用它。我們的價值到底在哪裡？我們怎麼和客戶一起創造價值？

**背景數據**：
- 平均售價：NT$3.5M / 台，毛利率 22%（5 年前是 31%）
- 售後服務佔營收 8%（保固期結束後流失率 74%）
- 客戶平均機台稼動率：58%（行業最佳實踐：85%）
- 現有顧客資料：僅有購買紀錄與型號，無使用行為資料

---

## Analysis

### Step 1 — 基礎前提稽核（Foundational Premise Audit）

| FP | 現況 | 問題癥結 |
|----|------|---------|
| FP1（服務是交換基礎）| 賣「機台」，訂單即結束 | 客戶真正需要的是「加工能力」，TM 把交付機台當作價值交付終點 |
| FP6（多方共創）| 客戶操作、TM 不介入 | TM 在客戶使用週期內缺席；稼動率 58% 意味大量共創機會浪費 |
| FP7（廠商只能提供價值主張）| 業務話術：「買我們的機台，你的良率就會提升」| 把價值內嵌在產品的思維——違反 FP7；良率是客戶操作員、夾治具、切削液整合的結果 |
| FP9（所有行動者都是資源整合者）| 不了解客戶帶來哪些資源（操作知識、材料、製程 know-how）| 無法設計互補性資源支持 |
| FP10（價值由受益者在使用情境中決定）| 以「機台規格」定義價值，而非「客戶生產良率或交期」| 指標脫節：TM 追售價，客戶追每片工件成本 |

**主導邏輯診斷**：強烈的商品主導邏輯（Goods-Dominant Logic）——價值被視為內嵌在機台規格中，交付即完成。

---

### Step 2 — 資源盤點：操作型 vs 被操作型

**TM 目前掌握的資源**

| 資源類型 | 項目 | 戰略地位 |
|---------|------|---------|
| 被操作型（Operand）| 鋼材、主軸、伺服馬達、庫存備件 | 已商品化，易被替代 |
| 操作型（Operant）| 40 年加工製程知識、刀具壽命預測模型（工程師腦中）、歐規認證 know-how、前 800 台機器的維修歷史紀錄 | **尚未貨幣化** |

**客戶帶進來的資源（TM 目前視而不見）**

| 資源類型 | 項目 |
|---------|------|
| 操作型 | 操作員製程技能、品管 SOP、與下游客戶的品質協議 |
| 被操作型 | 原材料、治具、廠房空間 |

**關鍵洞察**：TM 工程師腦中的「刀具壽命預測知識」是核心操作型資源，但目前以隱性知識存在，無法在客戶使用情境中發揮作用。

---

### Step 3 — 服務生態系地圖

```
Macro 層（制度安排）
  ├─ 歐盟 CE 認證 / 台灣 iA 智慧機械補助政策
  └─ 航太 AS9100 / 醫療 ISO 13485 認證壓力（驅動客戶升級需求）

Meso 層（服務系統）
  ├─ TM ──[機台 + 文件]──► 代理商 ──[安裝 + 教育訓練]──► 終端客戶
  ├─ 刀具供應商 ──[切削條件建議]──► 終端客戶    ← 目前 TM 缺席這個流
  └─ 客戶 ──[良率問題]──► Google / 論壇          ← 不找 TM

Micro 層（二元互動）
  現況：TM ↔ 客戶  僅在「購買」和「保固維修」兩個接觸點互動
  缺口：「日常操作」「製程優化」「換刀時機」——三個最高頻接觸點，TM 完全缺席
```

**資源流重新設計機會**：
- 將機台 IoT 感測數據（主軸電流、振動、溫度）轉換為可操作的製程建議，輸回給客戶操作員
- 將 800 台歷史維修記錄轉換為預測保養模型，由被動維修轉為主動介入

---

### Step 4 — 價值主張重新設計

| 舊主張（商品主導） | 新主張（服務主導） | 啟動資源 |
|-----------------|-----------------|---------|
| 「TM-V550 加工精度 ±0.002mm」 | 「讓您的航太件良率從 91% 提升到 96%，我們的製程工程師與您的操作員共同驗證」 | TM 製程工程師 + 客戶製程資料 |
| 「5 年保固，零件到位 48 小時」 | 「預測性保養訂閱：在刀具斷裂前 8 小時提醒，每年減少 3 次非計畫停機」 | 歷史維修資料 → 預測模型 |
| 「歐規 CE 認證機台」 | 「幫您的客戶通過 AS9100 加工能力審查，我們提供製程能力報告範本與顧問支持」 | TM 認證 know-how |

## Result

## S-D Logic Analysis: 台灣機械股份有限公司

### Current Logic Assessment
- Dominant logic: **強商品主導邏輯**（Goods-Dominant）
- Value definition: 純 value-in-exchange（訂單金額），未測量 value-in-use（客戶稼動率、每件加工成本）

### Foundational Premise Audit

| FP | 現況 | 缺口 | 行動 |
|----|------|------|------|
| FP1（服務基礎）| 賣機台，交貨即結束 | 「加工能力服務」才是客戶購買的本質 | 建立「製程訂閱」服務線 |
| FP6（共創）| TM 在使用週期缺席 | 58% 稼動率顯示大量共創價值浪費 | 部署 IoT + 製程工程師駐廠制 |
| FP10（受益者定義價值）| 以規格表定義產品價值 | 客戶 KPI 是「每件加工成本」非「主軸轉速」 | 將 SLA 重新定義為稼動率與良率目標 |

### Service Ecosystem Map
- **核心行動者**：TM 製程工程師、客戶操作員、刀具供應商（現缺席）、代理商（現為被動配送角色）
- **關鍵資源流**：機台感測數據（目前未採集）→ 製程建議 → 操作員行為改變 → 稼動率提升
- **制度安排**：善用台灣智慧機械補助政策（最高補助 NT$1,000 萬）降低客戶導入 IoT 的障礙

### Redesigned Value Propositions

1. **[From]** 一次性銷售 CNC 機台（NT$3.5M，毛利 22%）  
   **[To]** 「加工能力即服務」：機台 + 3 年製程訂閱（每月 NT$45,000），預計 5 年 LTV 提升 2.3 倍

2. **[From]** 被動保固維修（出問題才派人）  
   **[To]** 預測性保養訂閱：IoT 數據驅動，SLA 承諾「非計畫停機每年不超過 2 次」

3. **[From]** 交貨後由代理商提供 3 天教育訓練  
   **[To]** 持續製程共創：每季製程優化工作坊 + 線上即時諮詢，由 TM 工程師與客戶操作員共同驗證製程參數

### 警示（Co-destruction 風險）
若 TM 推出 IoT 訂閱但客戶工廠網路不開放外連，資源整合失敗將導致**共同價值毀損**（value co-destruction）——需先設計離線邊緣運算方案，確保資源整合條件成立再承諾 SLA。
