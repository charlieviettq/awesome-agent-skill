# Comparable Company Multiples (市場比較法)

DCF 的內在價值估算需要搭配市場比較法（Comparables）做交叉驗證。當 DCF 給出一個估值，你應該問：「市場上類似公司的交易倍數是否支持這個數字？」如果兩者差距超過 30%，必須解釋原因。

---

## 核心邏輯

市場比較法假設「相似的公司應有相似的估值倍數」。你找到一組可比公司（Peer Group），計算它們的估值倍數，再套用到目標公司的財務指標上。

```
估值倍數 = 企業價值（EV）或股票市值 / 某個財務指標
目標公司估值 = 目標公司財務指標 × 可比公司倍數（中位數）
```

---

## 常用倍數

### EV-based 倍數（適用於跨資本結構比較）

| 倍數 | 公式 | 最適用產業 |
|------|------|-----------|
| EV/EBITDA | EV ÷ EBITDA | 製造、電信、傳統零售 |
| EV/EBIT | EV ÷ EBIT | 資本輕的服務業 |
| EV/Revenue | EV ÷ Revenue | 高成長 SaaS、虧損公司 |
| EV/FCF | EV ÷ Free Cash Flow | 成熟現金牛企業 |

### Equity-based 倍數（直接得出股票價值）

| 倍數 | 公式 | 注意事項 |
|------|------|---------|
| P/E | 股價 ÷ EPS | 需要正盈利；受資本結構影響 |
| P/S | 市值 ÷ 營收 | SaaS 早期常用 |
| P/B | 股價 ÷ 每股淨資產 | 金融股、資產重型企業 |

### SaaS / 訂閱制專用

| 倍數 | 公式 | 說明 |
|------|------|------|
| EV/ARR | EV ÷ 年度經常性收入 | 最常見 SaaS 估值基礎 |
| EV/NRR | EV ÷ Net Revenue Retention | 反映擴張能力 |

**IRON LAW 連結**：EV/EBITDA 是 DCF 退出倍數法（Exit Multiple）的直接輸入。如果你用 EV/EBITDA = 12x 作為 DCF 的 Terminal Value，但市場上可比公司只交易在 8x，你的 Terminal Value 就高估了 50%。

## 步驟：建立可比公司倍數分析

### Step 1：篩選可比公司

可比公司必須在以下維度接近目標公司：

1. **產業**：相同的商業模式（不只是 SIC code）
2. **規模**：營收在目標公司的 0.3x–3x 之間
3. **成長性**：相近的收入成長率（±10 個百分點）
4. **地理市場**：如果是台灣公司，優先找台灣/亞太可比，再找全球
5. **獲利能力**：EBITDA margin 在相近區間

最終保留 **5–10 家**。少於 5 家統計意義不足；超過 10 家可能納入了不夠相似的公司。

### Step 2：收集財務數據

```
EV = 市值 + 淨負債
淨負債 = 總負債 - 現金及約當現金
```

資料來源優先順序：Bloomberg / FactSet → S&P Capital IQ → 公開財報 + 手動計算

**注意**：使用 **LTM（Last Twelve Months）** 或 **NTM（Next Twelve Months）** 需保持一致。成長股通常用 NTM，以避免高成長公司因歷史數字低而倍數虛高。

### Step 3：計算倍數分布

計算每家可比公司的倍數後，看分布：

| 統計量 | 用途 |
|--------|------|
| **中位數** | 主要估值錨點（比平均值更抗極端值） |
| 25th percentile | 保守估值下界 |
| 75th percentile | 樂觀估值上界 |
| 平均值 | 僅作參考；容易被一兩家極端值拉偏 |

### Step 4：套用倍數

```
目標公司 EV（中位數）= 目標公司 EBITDA × Peer Group EV/EBITDA 中位數
目標公司 Equity Value = EV - 目標公司淨負債
每股價值 = Equity Value ÷ 流通股數
```

## 實戰範例：台灣 SaaS 公司估值

**目標公司：** TaiwanSaaS Co.
- ARR：NT$500M，YoY 成長 25%
- EBITDA：NT$75M（margin 15%）
- 淨負債：NT$-80M（淨現金）
- 流通股數：50M 股

### 可比公司倍數表

| 公司 | EV（NTM）| ARR | EV/ARR | EBITDA | EV/EBITDA | ARR YoY |
|------|---------|-----|--------|--------|-----------|---------|
| PeerA | $2,400M | $200M | 12.0x | $30M | 80.0x | 30% |
| PeerB | $1,800M | $180M | 10.0x | $18M | 100.0x | 22% |
| PeerC | $900M | $120M | 7.5x | $24M | 37.5x | 18% |
| PeerD | $3,500M | $350M | 10.0x | $70M | 50.0x | 28% |
| PeerE | $600M | $80M | 7.5x | $8M | 75.0x | 15% |
| **中位數** | | | **10.0x** | | **75.0x** | |
| **25th pct** | | | **7.5x** | | **50.0x** | |
| **75th pct** | | | **10.0x** | | **80.0x** | |

> EV/EBITDA 倍數極端高（37.5x–100x）是因為這些公司仍在重投資期。對高成長 SaaS 應以 **EV/ARR** 為主要錨點。

### 估值計算

**以 EV/ARR（主要方法）：**

```
EV（中位數）= NT$500M × 10.0x = NT$5,000M
EV（25th）  = NT$500M × 7.5x  = NT$3,750M
EV（75th）  = NT$500M × 10.0x = NT$5,000M

Equity Value（中位數）= NT$5,000M - (-NT$80M) = NT$5,080M
每股價值（中位數）    = NT$5,080M ÷ 50M = NT$101.6
```

**估值區間：NT$75–NT$102（每股）**

### 與 DCF 的交叉驗證

假設 DCF 算出每股 NT$110：
- 差距 ≈ 8%，在合理範圍（<15%）✓
- 若 DCF 算出 NT$180，差距 63%：需重新檢視 DCF 的 Terminal Value 假設

## 倍數選擇決策樹

```
公司有正的 EBITDA？
├── Yes → 優先 EV/EBITDA，輔以 EV/Revenue
│         成長率 > 20%？
│         ├── Yes → 搭配 EV/ARR 或 EV/Revenue
│         └── No → EV/EBITDA 已足夠
└── No → 公司有穩定收入？
          ├── Yes → EV/Revenue（SaaS 用 EV/ARR）
          └── No → 慎用倍數法；改用 DCF 或 Venture Method
```

## 常見陷阱

### 1. 用不同時間基準混搭
PeerA 用 LTM，PeerB 用 NTM，算出來的倍數沒有可比性。**整組必須統一時間基準。**

### 2. 不調整非經常性項目
EBITDA 中包含一次性重組費用、訴訟和解金等，會讓倍數失真。應使用 **Adjusted EBITDA**，但要在報告中說明調整項目。

### 3. 台灣/亞太公司直接套用美國 SaaS 倍數
美國 SaaS 倍數在 2021 年高峰時達 30–40x ARR，2023 年回落至 5–10x。地理市場流動性、投資人偏好差異顯著。若找不到足夠的台灣可比公司，可用全球公司但加上 **折價 10–25%**（反映市場流動性差異）。

### 4. 把中位數當點估值
可比公司分析給出的是**範圍**，不是精確答案。永遠呈現 25th–75th percentile 的區間，而非只報告中位數。

### 5. 忽視目標公司與 Peer 的品質差距
目標公司 NRR 120%，但 Peer Group 平均 NRR 只有 95%？目標公司值得**溢價**，應套用倍數上緣或解釋溢價依據。反之亦然。

### 6. EV/EBITDA 用在高負債公司未標記
高負債公司的 EBITDA 倍數看起來低（因為高槓桿壓低 EV 中的 Equity 部分），可能造成「便宜」的假象。應同時呈現 P/E 或 EV/FCF 做確認。

---

## 與 DCF 的整合方式

| 情境 | 建議做法 |
|------|---------|
| DCF 與 Comparables 差距 < 15% | 以 DCF 為主，Comparables 作佐證 |
| 差距 15–30% | 重新審視兩者假設；可能需要加入 Premiums/Discounts |
| 差距 > 30% | 必須解釋差距來源（成長率假設？市場情緒？）；若無法解釋，報告應呈現兩個區間 |

**DCF Exit Multiple 一致性檢查：**
如果你的 DCF Terminal Value 使用 EV/EBITDA = 12x，但 Comparables 顯示市場倍數為 8x，你需要：
1. 降低 DCF 的退出倍數，或
2. 解釋為何目標公司在 5–10 年後應獲得溢價
