# Example: E-Commerce Checkout Flow A/B Test

## Scenario

MomoBuy（台灣電商平台）的產品分析師 Irene 跑來問：

> 「我們上個月測試了新版結帳流程，把三步驟改成一頁式。實驗組 2,400 人裡有 312 人完成購買，對照組 2,400 人有 264 人完成購買。換算下來轉換率從 11.0% 升到 13.0%，提升了 2 個百分點。但工程師說要等 p-value，請問這個結果顯著嗎？值得全量上線嗎？」

補充資訊：
- 實驗期間：2026-03-01 ~ 2026-03-28（28 天）
- 隨機分配：用戶 ID 哈希，各組 2,400 人
- 主指標：結帳完成率（付款成功 / 進入結帳頁）
- α 預設 0.05，實驗事前已登記（pre-registered）

---

## Analysis

### Step 1：確認假說（事前已登記，無 p-hacking 疑慮）

- **H₀**：新版一頁式結帳的完成率 = 舊版三步驟完成率（p₁ − p₀ = 0）
- **H₁**：新版完成率 > 舊版完成率（單尾，因為只關心「有沒有提升」）
- **α = 0.05**（標準閾值，事前設定）

### Step 2：選擇統計檢定

| 條件 | 本案 |
|------|------|
| 資料類型 | 二元（買 / 未買）→ 類別型 |
| 組數 | 2 組（實驗 vs. 對照） |
| 獨立性 | 用戶互相獨立（隨機分配） |

→ 選用 **Two-proportion Z-test（雙比例 Z 檢定）**
（樣本夠大：n₁p₁ = 312 > 5；n₀p₀ = 264 > 5，滿足常態近似條件）

### Step 3：計算

**原始數字：**

| 組別 | n | 轉換數 | 轉換率 |
|------|---|--------|--------|
| 對照組（舊版） | 2,400 | 264 | p₀ = 0.1100 |
| 實驗組（新版） | 2,400 | 312 | p₁ = 0.1300 |

**Pooled proportion（合併比例）：**

```
p̂ = (264 + 312) / (2400 + 2400) = 576 / 4800 = 0.1200
```

**Standard Error（標準誤）：**

```
SE = √[ p̂(1 − p̂)(1/n₀ + 1/n₁) ]
   = √[ 0.12 × 0.88 × (1/2400 + 1/2400) ]
   = √[ 0.1056 × 0.000833 ]
   = √0.00008800
   ≈ 0.009381
```

**Z 統計量（單尾）：**

```
Z = (p₁ − p₀) / SE
  = (0.1300 − 0.1100) / 0.009381
  = 0.0200 / 0.009381
  ≈ 2.132
```

**p-value（單尾）：**

```
P(Z > 2.132) ≈ 0.0165
```

**95% 信賴區間（雙尾，for reporting）：**

```
(p₁ − p₀) ± 1.96 × SE
= 0.0200 ± 1.96 × 0.009381
= 0.0200 ± 0.0184
= [0.0016, 0.0384]
```

**Effect size（Cohen's h）：**

```
h = 2 × arcsin(√p₁) − 2 × arcsin(√p₀)
  = 2 × arcsin(0.3606) − 2 × arcsin(0.3317)
  ≈ 2(0.3685) − 2(0.3382)
  ≈ 0.061
```

Cohen's h = 0.06 屬於 **small effect**（< 0.2 為 small）。

### Step 4：假說決策

- p-value = 0.0165 < α = 0.05 → **拒絕 H₀**
- 結果在統計上顯著

### Step 5：實際意義評估（IRON LAW 檢查）

> ⚠️ 統計顯著 ≠ 實際重要，必須評估效益規模

- 絕對提升：+2.0 個百分點（11% → 13%）
- 相對提升：+18.2%
- 信賴區間下界：+0.16 pp（最悲觀估計仍為正值）
- MomoBuy 每月進入結帳頁約 500,000 次
  → 年化增量訂單：500,000 × 12 × 0.020 ≈ **120,000 筆/年**
  → 若客單價 NT$1,200，則年增營收約 **NT$144,000,000**

效益明確，Cohen's h 雖小但絕對數字在商業上具重大意義。

## Result

```markdown
# Hypothesis Test: MomoBuy 一頁式結帳流程 A/B Test

## Hypotheses
- H₀: 新版結帳完成率 = 舊版結帳完成率（Δp = 0）
- H₁: 新版結帳完成率 > 舊版結帳完成率（Δp > 0）
- α = 0.05（單尾，事前設定）

## Test Selection
- Test: Two-proportion Z-test
- Rationale: 二元結果變數、兩個獨立樣本、n 夠大符合常態近似
- Assumptions checked:
  - 獨立性：✅ 用戶 ID 哈希隨機分配，各組互不重疊
  - 樣本量：✅ np > 5 for all cells（264, 2136, 312, 2088）
  - 無重複曝光：✅ 每位用戶僅分配一次

## Results
- Test statistic: Z = 2.132
- p-value: 0.0165（單尾）
- Effect size: Cohen's h = 0.06（small）
- 95% CI for Δp: [+0.16%, +3.84%]
- Absolute lift: +2.0 pp（11.0% → 13.0%）

## Decision
**拒絕 H₀**（p = 0.0165 < α = 0.05）

## Interpretation
新版一頁式結帳在統計上顯著優於舊版（p = 0.0165）。雖然
Cohen's h = 0.06 屬於小效應，但 2 pp 的絕對提升在 MomoBuy
的流量規模下，估計可帶來約 NT$144M 的年增營收。95% CI 下界
仍為正值（+0.16 pp），顯示即使最保守的估計仍有正面效益。

建議全量上線，並在上線後 28 天持續監測轉換率是否落在
[12.16%, 14.84%]（95% CI），以確認實驗結果可複製。
```

**補充：常見誤讀提醒給 Irene**

1. **p = 0.0165 不代表「有 98.35% 機率新版比較好」**——p-value 是「假設 H₀ 為真，看到這筆資料的機率」，不是 H₀ 為真的機率（base rate fallacy）。

2. **不需要 Bonferroni 修正**——本實驗只有一個主指標，無多重比較問題。若同時也測了「平均訂單金額」等次要指標，那些的 α 需要額外調整。

3. **「fail to reject H₀ ≠ H₀ 為真」**——若這次 n = 200 而非 2,400，power 不足可能看不到同樣 2 pp 的差異，但差異依然存在。
