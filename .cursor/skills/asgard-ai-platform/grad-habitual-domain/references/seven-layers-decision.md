# 七層決策結構（Seven Layers of Decision）

> 資料來源：Yu (1985, 1990, 2010)、游伯龍（1998, 2008）

## 結構總表

游伯龍將一個決策問題的元素分解為七個層次。**注意：這是「決策元素的分層」，不是「決策流程的階段」**。Simon 的 Intelligence-Design-Choice 才是流程；七層是「決策結構的切片」。

```
┌───────────────────────────────────────────────┐
│ Layer 7：決策者自我狀態（Ego State）             │
│   情緒、疲勞、價值信念、生理狀態                  │
├───────────────────────────────────────────────┤
│ Layer 6：資訊情境（Information Situation）        │
│   擁有多少、品質、對稱性、及時性                  │
├───────────────────────────────────────────────┤
│ Layer 5：偏好結構（Preference Structure）          │
│   權重、取捨、風險偏好                          │
├───────────────────────────────────────────────┤
│ Layer 4：結果評估（Outcome Evaluation）            │
│   方案的可能結果與機率                          │
├───────────────────────────────────────────────┤
│ Layer 3：準則／目標（Criteria/Goals）              │
│   評估指標、績效衡量                            │
├───────────────────────────────────────────────┤
│ Layer 2：方案集合（Alternative Set）               │
│   目前考慮的選項                                │
├───────────────────────────────────────────────┤
│ Layer 1：參數辨識（Parameter Identification）      │
│   問題是什麼、邊界、變數                        │
└───────────────────────────────────────────────┘
```

---

## 各層次詳解

### Layer 1：參數辨識
**關鍵提問**：「這個決策到底在回答什麼問題？」

**常見錯置**
- 錯把症狀當問題（「我們業績下滑」是症狀，問題可能是「產品生命週期」或「通路失效」）
- 邊界設得太窄（只看內部，忽略法規或競爭者變動）
- 變數漏掉（定價決策只看成本與競爭者，忽略顧客知覺價值）

**EMBA 作業品質標誌**：在作業開頭花 1 頁釐清問題定義，通常就贏了一半。

---

### Layer 2：方案集合
**關鍵提問**：「我們還漏了哪些方案？」

**常見病態**
- 二元陷阱：「做 vs. 不做」（通常至少還有第三條路）
- 現狀錨定：把「維持不變」視為預設，不列入評估
- 顧問偏食：只評估外部顧問推薦的方案

**擴展路徑（呼應 HD 擴展工具）**
- 參數變動：若預算翻倍、時間減半？
- 類比：同業、跨域、國外做法？
- 逆向：有哪些方案是「禁止考慮」的？為何？

---

### Layer 3：準則／目標
**關鍵提問**：「我們用什麼衡量成功？」

**常見誤用**
- 單一指標崇拜：只看 ROI 不看策略對齊
- 指標衝突：追求市占率同時要求短期獲利（互斥）
- 無法觀測的準則：「提升文化」無法量測 → 必須轉換為可觀測代理指標

**EMBA 常見加分**：明確區分「必要準則（must）」、「滿足準則（should）」、「加分準則（nice-to-have）」。

---

### Layer 4：結果評估
**關鍵提問**：「每個方案的可能結果與機率？」

**處理工具**
- 決策樹（Decision Tree）
- 情境規劃（Scenario Planning）
- 敏感度分析（Sensitivity Analysis）

**HD 觀點的補充**
- 結果評估受 Activated Domain 限制——人只評估想得到的結果
- 常見盲點：忽略 tail risk（低機率高衝擊）、忽略綜效與負外部性

---

### Layer 5：偏好結構
**關鍵提問**：「當準則互相衝突時，如何取捨？」

**偏好類型**
- 字典序（Lexicographic）：先看 A，A 同則看 B
- 補償式（Compensatory）：A 弱但 B 強可抵銷
- 門檻式（Threshold）：不達標一律淘汰

**EMBA 常見錯誤**
- 不說明偏好結構，直接給「綜合評比」分數
- 偏好權重拍腦袋決定，缺少決策者對話
- 忽略偏好會隨情境改變（Layer 7 影響 Layer 5）

---

### Layer 6：資訊情境
**關鍵提問**：「我們手上的資訊，品質如何？有什麼偏差？」

**檢核清單**
- 資訊來源是否獨立？
- 是否存在資訊不對稱（如內部人 vs. 外部董事）？
- 資料時效性（三年前的市場調查可用嗎）？
- 是否有刻意隱瞞或誤導的誘因？

**台灣 EMBA 情境常見問題**
- 過度依賴單一媒體（工商時報 / 財訊）的觀點
- 忽略內部一線員工與高階的資訊落差
- 國際資訊用 Google 翻譯一手，失真

---

### Layer 7：決策者自我狀態
**關鍵提問**：「決策者現在的狀態適合做這個決策嗎？」

**子層次**
- 生理（睡眠、健康、時差）
- 情緒（憤怒、焦慮、過度自信）
- 價值信念（意識形態、宗教、世代觀）
- 身分認同（家族、宗族、階級位置）

**常見盲點**
- 董事會視決策者為「理性人」，忽略 Layer 7
- 併購決策在疲勞狀態下拍板 → 後悔常源此
- 家族企業決策摻雜個人恩怨（Layer 7 主導 Layer 5）

**EMBA 口試加分題**：能指出「決策者 X 的 Layer 7 狀態如何影響其 Layer 2 選擇」，通常是優等分數的標誌。

---

## 七層間的互動（Cross-Layer Dynamics）

```
Layer 7（自我狀態）─┬→ 影響 Layer 5（偏好）
                   ├→ 影響 Layer 3（準則）
                   └→ 影響 Layer 1（問題定義）
Layer 6（資訊）───┬→ 決定 Layer 4（結果評估）
                  └→ 約束 Layer 2（方案集合）
Layer 1 → Layer 2 → Layer 3 → Layer 4 → Layer 5 最終整合
```

**關鍵洞見**：下層（Layer 1-2）看似技術性，其實深受上層（Layer 6-7）污染。若分析卡在下層換個方案，問題常出在上層需先解。

---

## 個案分析工作範本

**Step 1**：逐層盤點案文中決策者的狀態
- Layer 1-7 各寫 2–3 句描述

**Step 2**：指出主要卡點（通常 1–2 層）
- 用案文證據支持

**Step 3**：連結 HD 通性
- 卡在 Layer 2 常因通性 7（固化）+ 5（暈輪）
- 卡在 Layer 7 常因通性 4（迴避）+ 1（循環）

**Step 4**：設計介入
- 技術性卡點（Layer 1-4）→ 結構化工具（決策樹、清單）
- 心理／資訊卡點（Layer 5-7）→ HD 擴展工具（投射、參數變動、外部擾動）

## 與其他決策模型的區別

| 模型 | 本質 | 與七層關係 |
|---|---|---|
| Simon IDC | 流程階段（找問題→設計→選擇） | 七層是 IDC 每階段內的結構 |
| Kepner-Tregoe | 問題分析 + 決策分析工具 | KT 較聚焦 Layer 1-3，七層更廣 |
| Herbert A. Simon Bounded Rationality | 理性有上限 | 七層是 BR 的結構化展開 |
| Vroom-Yetton | 領導者何時讓下屬參與決策 | 與七層互補（V-Y 看參與度，七層看結構） |
| AHP | 層級分析 + 權重量化 | AHP 是 Layer 3-5 的工具，不處理 Layer 6-7 |

**結論**：七層決策結構的獨特性在於**同時涵蓋技術層與心理／資訊層**，是少數能將「決策者自我狀態」正式納入結構的決策理論。
