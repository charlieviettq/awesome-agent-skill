# Operating Model 重構與敏捷組織

## 六大維度

DX 不是單純技術專案，真正的轉型必須同時重構營運模式的六大維度。

### 1. 組織架構
- 事業部 vs. 職能 vs. 矩陣
- 獨立 DX 事業群 vs. 整合
- 敏捷部落（Tribe / Squad 模式）

### 2. 人才
- 數位人才招募
- 既有人才再培訓（Reskilling）
- 人才流動與輪調
- 員工數位素養

### 3. 流程
- 端到端流程重設計
- 自動化優先
- 跨部門流程擁有者

### 4. 資料
- 資料治理（CDO）
- 資料平台
- 資料文化

### 5. 技術
- 雲端優先
- API 優先
- DevOps / DevSecOps

### 6. 文化與管理方式
- 決策速度
- 失敗容忍
- 客戶中心
- 資料驅動

## 敏捷組織（Agile Organization）

### Spotify 模型
- **Squad**：跨功能小隊（8 人以下、自主產品）
- **Tribe**：多 Squad 組成的部落（同領域）
- **Chapter**：同職能跨 Squad 的橫向專業社群
- **Guild**：跨部落興趣社群

**適合**：數位產品、軟體、數據團隊
**不適合**：製造業產線、合規繁重業務

**台灣企業常見錯誤**：直接照搬 Spotify 模型、未調整本土文化。建議「**部分敏捷**」：核心數位團隊採敏捷、其他維持傳統。

### 切換敏捷的前提
- 領導層真正願意授權
- 失敗容忍度顯性提高
- 員工有獨立決策能力
- 客戶需求變動快

### 常見誤用
- 形式敏捷：用 Jira 開站立會議，但決策仍在上層
- 跨部門不跨功能：Squad 內缺少真正端到端能力
- 沒有產品負責人（PO）制度
- KPI 仍按職能設定，而非 Squad 產出
