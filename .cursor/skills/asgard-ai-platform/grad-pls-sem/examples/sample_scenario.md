# Example: 行動支付採納意圖之 PLS-SEM 分析

## Scenario

**研究者**：台灣某國立大學管理學院博士生，正在撰寫關於行動支付使用意圖的論文。

**研究問題**：哪些因素影響台灣中小企業主採納行動支付（如 LINE Pay、街口支付）？

**研究模型**：
- 外生構念（3 個）：**感知有用性**（PU，reflective，4 題）、**感知易用性**（PEOU，reflective，3 題）、**服務品質**（SQ，formative，4 題）
- 中介構念（1 個）：**使用態度**（ATT，reflective，3 題）
- 內生構念（1 個）：**採納意圖**（BI，reflective，3 題）

**問題**：「我的樣本只有 N = 143，指導教授說不夠跑 CB-SEM，建議改用 PLS-SEM。我用 SmartPLS 4 跑完了，但不確定怎麼判讀結果、哪些數字要報告、以及服務品質要怎麼處理（它是 formative 的）。」

**資料特徵**：N = 143（台灣中小企業主問卷調查），Likert 7 點量表，非常態分布（Skewness 多數 > 1.0）。

---

## Analysis

### Step 1 — 確認模型規格合理性

**為何選 PLS-SEM？**
- N = 143；指向 BI 的路徑最多 1 條（ATT → BI），指向 ATT 的路徑有 3 條（PU、PEOU、SQ → ATT）。PLS 最低門檻 = 10 × 3 = 30，143 遠超過，但對 CB-SEM 而言仍偏小。
- 模型含 formative 構念（SQ）：CB-SEM 無法直接估計 formative model。
- 研究目標是預測與解釋變異（R²），非嚴格理論驗證（不需 CFI/RMSEA）。

**構念規格**：
- PU、PEOU、ATT、BI → Reflective（指標反映潛在概念；移除一題不改變構念意義）
- SQ → Formative（四個面向共同構成服務品質：系統穩定性、回應速度、隱私安全、客服支援；缺少任一面向即改變構念）

---

### Step 2 — Reflective 測量模型評估

SmartPLS 4 bootstrapping（5,000 samples）結果：

**指標信度（Factor Loadings）**

| 構念 | 指標 | Loading | t-value |
|------|------|---------|---------|
| PU | PU1 | 0.82 | 18.4 |
| PU | PU2 | 0.79 | 15.2 |
| PU | PU3 | 0.85 | 22.1 |
| PU | PU4 | 0.71 | 10.3 |
| PEOU | PEOU1 | 0.88 | 26.7 |
| PEOU | PEOU2 | 0.84 | 19.8 |
| PEOU | PEOU3 | 0.78 | 14.1 |
| ATT | ATT1 | 0.91 | 38.2 |
| ATT | ATT2 | 0.87 | 29.5 |
| ATT | ATT3 | 0.83 | 21.6 |
| BI | BI1 | 0.93 | 47.3 |
| BI | BI2 | 0.89 | 31.4 |
| BI | BI3 | 0.86 | 24.8 |

所有 loadings ≥ 0.70 ✓（最低 PU4 = 0.71，邊緣但可接受）

**收斂效度與信度**

| 構念 | CR | AVE | Cronbach's α |
|------|----|-----|--------------|
| PU | 0.884 | 0.658 | 0.836 |
| PEOU | 0.880 | 0.709 | 0.797 |
| ATT | 0.912 | 0.776 | 0.856 |
| BI | 0.928 | 0.811 | 0.883 |

CR ≥ 0.70 ✓，AVE ≥ 0.50 ✓（收斂效度成立）

**區別效度（HTMT）**

| | PU | PEOU | ATT | BI |
|--|-----|------|-----|-----|
| PU | — | | | |
| PEOU | 0.61 | — | | |
| ATT | 0.74 | 0.68 | — | |
| BI | 0.72 | 0.65 | 0.81 | — |

所有 HTMT < 0.90 ✓（區別效度成立）

> **注意**：此處使用 HTMT 而非 Fornell-Larcker，因 PLS-SEM 中 HTMT 對區別效度更為敏感且建議優先使用（Henseler et al., 2015）。

---

### Step 3 — Formative 測量模型評估（SQ）

**指標共線性（VIF）**

| 指標 | 內容 | VIF |
|------|------|-----|
| SQ1 | 系統穩定性 | 2.34 |
| SQ2 | 回應速度 | 1.87 |
| SQ3 | 隱私安全 | 2.51 |
| SQ4 | 客服支援 | 1.62 |

所有 VIF < 5.0 ✓（無嚴重共線性問題）

**指標權重（Bootstrapping, 5,000 samples）**

| 指標 | Weight | t-value | p-value | 顯著？ |
|------|--------|---------|---------|--------|
| SQ1（系統穩定性）| 0.38 | 4.21 | < 0.001 | ✓ |
| SQ2（回應速度） | 0.29 | 2.87 | 0.004 | ✓ |
| SQ3（隱私安全） | 0.41 | 4.56 | < 0.001 | ✓ |
| SQ4（客服支援） | 0.18 | 1.73 | 0.084 | ✗ |

SQ4 weight 未達顯著（p = 0.084）。**建議**：保留 SQ4，因為「客服支援」在理論上是服務品質的必要面向，formative 指標不應僅因統計不顯著而刪除；但需在論文中說明並以 loading 輔助（SQ4 loading = 0.62，對構念仍有相關）。

---

### Step 4 — 結構模型評估

**路徑係數與假設檢定**（Bootstrapping 95% CI）

| 假設 | 路徑 | β | t-value | p-value | 95% CI | f² | 支持？ |
|------|------|---|---------|---------|---------|-----|--------|
| H1 | PU → ATT | 0.31 | 4.12 | < 0.001 | [0.16, 0.46] | 0.12 | ✓ |
| H2 | PEOU → ATT | 0.24 | 3.38 | 0.001 | [0.10, 0.38] | 0.07 | ✓ |
| H3 | SQ → ATT | 0.28 | 3.74 | < 0.001 | [0.13, 0.43] | 0.09 | ✓ |
| H4 | ATT → BI | 0.67 | 12.84 | < 0.001 | [0.56, 0.78] | 0.81 | ✓ |

f² 解讀：0.02 = small, 0.15 = medium, 0.35 = large（Cohen, 1988）
- H4（ATT → BI）f² = 0.81，大效果 ✓
- H1（PU → ATT）f² = 0.12，接近中等效果
- H3 與 H2 為小至中等效果

**模型品質**

| 內生構念 | R² | Q²（Blindfolding, d=7）|
|---------|-----|----------------------|
| ATT | 0.548 | 0.387 |
| BI | 0.449 | 0.351 |

- ATT R² = 0.548（中至高度解釋力）；BI R² = 0.449（中度）
- Q² > 0 → 兩個內生構念均具預測相關性 ✓

```markdown
### Limitations
- 研究目標為預測解釋，不提供整體模型適配指數（CFI、RMSEA）；
  若需理論驗證，應改用 CB-SEM 並確保足夠樣本數
- SQ 為 formative 構念，無法以傳統 AVE/CR 評估；
  需額外進行 content validity 審查
- 橫斷面資料無法推論因果方向
```

**給研究者的關鍵建議**：
1. 論文中清楚說明選擇 PLS-SEM 的三個理由：小樣本、含 formative 構念、預測導向目標
2. 不要在論文中報告 CFI/RMSEA，這是 CB-SEM 的指標，用在 PLS 結果上是方法論錯誤
3. SQ4 保留決策需在論文中以一段說明理論依據，避免審查委員質疑
4. Bootstrapping 結果需明確標注 5,000 resamples，並報告 95% bias-corrected CI
