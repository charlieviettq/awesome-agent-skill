# Example: 寵物用品電商新創的台灣金流整合

## Scenario

**Pawsome Taiwan** 是一家剛完成產品開發的寵物用品電商，準備在 2026 年 5 月正式上線。創辦人 Lisa 詢問：

> 「我們的目標客群是 25-45 歲養貓狗的台灣消費者，預計月交易量 300-500 筆，客單價約 NT$800-2,000。我需要決定接哪些付款方式、選哪家金流商，以及大概要花多少時間整合。技術團隊只有一個全端工程師，我不想弄太複雜。」

## Analysis

### Step 1：確認付款方式組合

依據台灣電商 IRON LAW，最少要支援三種付款方式。根據 Pawsome 的客群（25-45 歲、中高消費力），對應市場分佈：

| 方式 | 台灣市佔 | Pawsome 客群適配度 | 決策 |
|------|---------|-----------------|------|
| 信用卡 | ~45% | 高（主力客群持卡率高）| **納入** |
| ATM 虛擬帳號 | ~15% | 中（部分不習慣刷卡者）| **納入** |
| 超商代碼 | ~10% | 中低（客單價 NT$800+ 用超商較少）| **納入**（保留選項） |
| LINE Pay | ~12% | 高（台灣 LINE 滲透率 95%）| **納入** |
| 超取付款 | ~8% | 低（寵物飼料/用品體積大，較少超取）| 暫不納入 |
| JKoPay | ~3% | 低 | 暫不納入 |

**最終選擇：信用卡 + ATM + 超商代碼 + LINE Pay（共 4 種）**

---

### Step 2：PSP 選擇

Pawsome 的條件：
- 月交易量 300-500 筆，客單價平均 NT$1,200 → 月交易額約 NT$36-60 萬
- 只有一位工程師，希望整合工時最短
- 需要信用卡、ATM、CVS、LINE Pay 一次到位

| PSP | 月費用試算（500 筆，均價 NT$1,200）| 整合難度 | LINE Pay |
|-----|-------------------------------|---------|----------|
| 綠界 ECPay | 信用卡：2.75% × NT$540,000 = **NT$14,850**；ATM：NT$15×150筆 = **NT$2,250** | 文件完整但較舊 | ✅ 支援 |
| 藍新 NewebPay | 信用卡：2.8% × NT$540,000 = **NT$15,120** | API 設計現代 | ✅ 支援 |
| TapPay | 2.8% × NT$540,000 = **NT$15,120** | Mobile-first | ✅ 原生支援 |

**推薦：綠界 (ECPay)**

理由：
1. 涵蓋信用卡、ATM、CVS、LINE Pay 單一帳號全搞定，工程師不需串接多家
2. 台灣電商整合範例最多，StackOverflow / GitHub 討論資源豐富，一人工程師排錯成本低
3. 月費用比藍新低 NT$270，一年省 NT$3,240（規模小時差距不大，但無痛切換）

---

### Step 3：整合流程確認

標準流程（以綠界為例）：

```
1. Lisa 的後台建立訂單 → 呼叫 ECPay AIO API (CreateTransaction)
2. ECPay 回傳 HTML form（含 CheckMacValue）
3. 前端 auto-submit 表單 → 跳轉至 ECPay 付款頁
4. 消費者選擇付款方式並完成付款
5. ECPay 打 Callback URL（POST）到 Pawsome 後台
6. 後台驗證 CheckMacValue（HMAC-SHA256）→ 更新訂單狀態
7. 回傳 "1|OK" 給 ECPay 確認收到
8. 消費者被導回 Return URL（感謝頁）
```

---

### Step 4：退款流程設計

Pawsome 的退款政策是 7 天內可申請退貨：

| 付款方式 | 退款方式 | 時間 | CS 處理成本 |
|---------|---------|------|-----------|
| 信用卡 | 呼叫 ECPay DoAction（退款 API）| 1-7 個工作天 | 低 |
| ATM 虛擬帳號 | **無法自動退款**，需向客戶收集銀行帳號，手動匯款 | 3-7 天 | 高 |
| 超商代碼 | **無法自動退款**，同上 | 3-7 天 | 高 |
| LINE Pay | 呼叫 LINE Pay Refund API | 1-3 個工作天 | 低 |

**建議**：在結帳頁面加註「ATM/超商付款退款需提供銀行帳號，處理時間 5-7 個工作天」，降低客戶期望落差。

---

### Step 5：安全清單確認

| 項目 | Pawsome 狀態 | 行動 |
|------|------------|------|
| HTTPS | 需確認 | 申請 SSL 憑證（Let's Encrypt 免費）|
| CheckMacValue 驗證 | 待實作 | Webhook handler 必須先驗簽再處理 |
| Idempotency | 待設計 | 以 `TradeNo` 為 key，重複 callback 直接回 "1\|OK" 不重複處理 |
| PCI DSS | 不需要 | 使用 ECPay 託管頁面，不碰原始卡號，免 PCI 認證 |
| 3D Secure | ECPay 預設啟用 | 保持開啟，降低詐刷風險 |

---

### 風險提醒

1. **商家審核時程**：ECPay 信用卡商家申請需 1-3 週（需公司登記謄本、銀行存摺封面）。若 5 月上線，最晚 **4 月 10 日**就要送件。
2. **ATM 退款流程**：Lisa 需在客服系統（不管用什麼 helpdesk）加一個「退款帳號收集」欄位，否則退款會卡在 email 來回。
3. **沙箱測試**：ECPay 沙箱與正式環境的 `MerchantID`、`HashKey`、`HashIV` 不同，務必在上線前確認環境切換正確，這是最常見的上線當天 bug。
