# Example: SME 線上借貸平台的逆選擇危機

## Scenario

CreditBridge（信橋金融）是一家台灣中小企業線上借貸平台，2024 年為搶攻市占率將年利率從 12% 降至 9%。六個月後，逾期 90 天以上的壞帳率從 4.2% 跳升至 9.8%，獲利由正轉負。風控長懷疑是「客群品質下降」，但不清楚是利率調整造成逆選擇，還是景氣問題。她需要診斷市場失靈的性質，並設計修正機制。

背景數據：
- 放款組合：2,400 家企業，平均貸款 NT$1,200,000
- 資金成本：6%；平台管理費：1.5%
- 低風險借款人（佔母體約 40%）：預期年違約率 1.5%，願付利率上限 10%
- 高風險借款人（佔母體約 60%）：預期年違約率 11%，願付利率上限 14%
- 9% 利率下實際違約率：9.8%（暗示客群幾乎全為高風險）

## Analysis

### Step 1 — 識別資訊不對稱

**不對稱類型**：事前逆選擇（hidden type）。中小企業主了解自身現金流健康程度、訂單可見度與還款意願；CreditBridge 的徵信模型僅觀察到財報、信用分數等落後指標，無法區分低風險與高風險借款人。

是否涉及道德風險？貸後監控數據顯示，部分借款人取得資金後降低其他融資渠道的還款優先順序（資金挪用），確認道德風險也存在，但主要問題是逆選擇——壞帳率在放款後第 1–2 個月即顯著升高，符合事前客群選擇問題，非事後行為問題。

### Step 2 — 建構市場失靈模型

**混同均衡（Pooling）分析：**

若 CreditBridge 以單一利率 9% 服務全體客群：

```
期望損失率 = 0.40 × 1.5% + 0.60 × 11% = 0.6% + 6.6% = 7.2%
最低盈虧平衡利率 = 資金成本 6% + 管理費 1.5% + 損失 7.2% = 14.7%
```

9% 遠低於 14.7%，混同定價無法持續。

**市場解體動態（Unraveling）：**

| 輪次 | 客群組成（低：高） | 加權違約率 | 盈虧平衡利率 | 動態 |
|------|-----------------|-----------|------------|------|
| 初始  | 40% : 60%       | 7.2%      | 14.7%      | 9% 虧損 |
| ↓ 調升至 10% | 低風險退場（願付上限 10%）→ 20% : 80% | 9.2% | 16.7% | 惡化 |
| ↓ 調升至 12% | 低風險全退 → 0% : 100% | 11.0% | 18.5% | 市場崩潰 |

阿克洛夫解體效應成立：每次漲價都驅逐更多低風險借款人，迫使再次漲價，形成死亡螺旋。

### Step 3 — 評估修正機制

**機制 A：利率菜單篩選（Rothschild-Stiglitz Screening）**

設計兩層合約誘導自我揭示：

- **A 方案（低風險型）**：年利率 8.5%、貸款上限 NT$2,000,000、需提供應收帳款擔保
- **B 方案（高風險型）**：年利率 13.5%、貸款上限 NT$800,000、無需擔保

激勵相容條件（IC）：
- 低風險借款人選 A 方案的效用 > 選 B 方案：擔保成本對低風險者小（違約概率低，擔保幾乎不會被執行）
- 高風險借款人選 B 方案的效用 > 選 A 方案：擔保成本對高風險者大（違約後擔保損失高），且貸款上限對急需資金者設有誘因妥協

參與約束（PC）：
- 低風險者：8.5% < 願付上限 10% ✓
- 高風險者：13.5% < 願付上限 14% ✓

分離均衡下的預期獲利：
```
低風險組：(8.5% - 6% - 1.5% - 1.5%) × 40% = -0.5% × 40% = -0.2%（略虧，但留住好客戶）
高風險組：(13.5% - 6% - 1.5% - 11%) × 60% = -5% × 60% = -3.0%（若無差別仍虧）
```

高風險組利率需更高或貸款規模更小；以 NT$800K × 13.5% 計算，違約損失降為 11% × 800K = NT$88K，vs. 利息收入 108K，邊際可行。

**機制 B：強制揭露 + 第三方徵信增強**

要求借款人提供稅務 API 授權（串接財政部電子發票系統），使 CreditBridge 能即時觀察月營收，將資訊不對稱從「隱藏類型」轉化為「可驗證類型」：

- 預估可識別 60–70% 的低風險借款人（月收入穩定、應稅收入高）
- 維持隱私保護（僅讀取彙總數字，非明細）
- 成本：每案 NT$300 的 API 查詢費，可納入申辦費

**機制 C：聲譽與重複交易**

對 12 個月正常還款的借款人提供「信譽等級」降息（下調 1%），形成動態篩選。高風險借款人由於預期違約概率高，對未來降息的折現價值低，不願意「模仿」低風險者維持良好還款紀錄——符合單交叉條件。

### Step 4 — 效率評估與政策建議

**全資訊基準（First-Best）：**
- 低風險者獲得利率 9%（資金成本 6% + 管理費 1.5% + 損失 1.5%）、貸款無上限
- 高風險者獲得利率 18.5%（遠超願付上限 14%），市場不服務此群體
- 社會最適：僅低風險市場存在，無資源浪費

**福利損失來源：**

| 損失類型 | 說明 | 估算影響 |
|---------|------|---------|
| 缺失交易 | 好客戶因高利率退出（每年約 960 家企業） | 消費者剩餘損失 NT$5.8M |
| 篩選扭曲 | 高風險組貸款上限壓縮至 NT$800K（低於需求） | 效率損失 NT$2.1M |
| 道德風險殘餘 | 擔保要求無法完全覆蓋貸後行為問題 | 增加監控成本 NT$1.2M |

**目前無管制混同均衡**：年損失約 NT$43M（壞帳 9.8% × 放款規模）。

## Result

## Information Economics Analysis: CreditBridge SME Lending

### Information Structure
- **Asymmetry type**: Adverse selection（主要）+ Moral hazard（次要）
- **Informed party**: 借款中小企業（知道自身真實現金流與還款意願）
- **Uninformed party**: CreditBridge（僅有落後財務指標）
- **Hidden variable**: 企業真實違約風險類型（低風險 1.5% vs. 高風險 11%）

### Market Failure Diagnosis
- **Unraveling risk**: 高——利率若維持 9%，市場將在 2–3 輪調整後崩潰
- **Pooling outcome**: 14.7% 才能盈虧平衡，遠超低風險借款人願付上限，市場不可維持
- **Separating outcome**: 利率菜單（8.5% + 擔保 vs. 13.5% + 小額）可還原部分效率

### Corrective Mechanisms
| Mechanism | Who Initiates | How It Works | Effective? |
|-----------|---------------|--------------|------------|
| 利率菜單篩選 | Uninformed（平台）| 擔保要求降低低風險成本，分離兩類客群 | 高，但低風險組微幅補貼 |
| 稅務 API 徵信 | Uninformed（平台）| 將隱藏類型轉為可驗證資訊 | 高，降低逆選擇根源 |
| 信譽等級降息 | Informed（借款人）| 重複交易形成自然分離 | 中，需 12 個月觀察期 |
| 強制財務揭露 | Government | 標準化財報提交，降低全市場資訊不對稱 | 中長期有效，短期成本高 |

### Efficiency Assessment
- **Full-information benchmark**: 服務全部低風險企業（960 家），利率 9%；不服務高風險
- **Welfare loss sources**: 缺失交易（低風險客戶被排擠）NT$5.8M + 篩選扭曲 NT$2.1M + 監控成本 NT$1.2M = NT$9.1M/年
- **Net welfare**: 利率菜單 + 稅務 API 組合可將年損失從 NT$43M 降至估計 NT$12M（改善 72%）

### Recommendation

**立即行動（Q1）：** 推出雙層利率菜單，停止單一 9% 定價。高風險層定價 13–14%、貸款上限 NT$800K；低風險層定價 8.5%、上限 NT$2M、需應收帳款擔保或稅務 API 授權。

**中期行動（Q2–Q3）：** 與財政部電子發票 API 串接，將徵信從落後財報升級為即時現金流監控；對 12 個月正常客戶啟動信譽降息計畫。

**政策倡議：** 若金管會推動 SME 融資資訊揭露標準化（類似美國 Small Business Lending Disclosure），可大幅降低全市場逆選擇，CreditBridge 應積極參與制度設計，搶占規範制定先機。

> **注意**：稅務 API 篩選若過度依賴發票收入，將系統性排除「低申報、高實際現金流」的傳統產業（如批發商），需保留人工覆核通道，避免因信息不對稱轉移而產生新的排除型偏誤。
