# Visual Specs — 各平台視覺規格

各平台演算法對視覺格式有明確偏好，錯誤的尺寸或格式會直接導致裁切、降觸及、或被標記為低品質內容。

---

## Instagram

### 圖片格式與尺寸

| 格式 | 尺寸（px） | 長寬比 | 備注 |
|------|-----------|--------|------|
| 正方形 Feed | 1080 × 1080 | 1:1 | 最安全，縮圖顯示完整 |
| 直式 Feed | 1080 × 1350 | 4:5 | **佔用動態牆最大版面，觸及率較高** |
| 橫式 Feed | 1080 × 566 | 1.91:1 | 適合風景、橫幅，縮圖較小 |
| Stories / Reels 封面 | 1080 × 1920 | 9:16 | 下方 250px 為 UI 覆蓋區 |
| Reels 影片 | 1080 × 1920 | 9:16 | 最長 90 秒，建議 15-30 秒 |
| Carousel 單張 | 1080 × 1080 或 1080 × 1350 | 同 Feed | 同輯保持同一比例 |

### 安全區（Safe Zone）

Stories 和 Reels 的頂部與底部各有 UI 元素覆蓋：

```
┌─────────────────────┐ ← 頂部 ~250px：帳號名稱、進度條
│   ⚠ 避免置放文字    │
├─────────────────────┤
│                     │
│   ✅ 安全創作區域   │ ← 中間 ~1420px
│   (文字/主視覺)     │
│                     │
├─────────────────────┤
│   ⚠ 避免置放文字    │ ← 底部 ~250px：按讚、留言、分享鈕
└─────────────────────┘
```

### 影片規格

| 項目 | Feed 影片 | Reels | Stories |
|------|-----------|-------|---------|
| 格式 | MP4 / MOV | MP4 / MOV | MP4 / MOV |
| 最大檔案 | 100 MB | 1 GB | 4 GB |
| 幀率 | 23-60 fps | 23-60 fps | 23-60 fps |
| 建議碼率 | 3,500 kbps | 5,000+ kbps | 3,500 kbps |
| 字幕 | 強烈建議（85% 靜音播放） | 強烈建議 | 可選 |

### Carousel 設計原則

- **第一張決定點擊率**：用視覺懸念（如「→ 滑動看完整清單」）
- **風格一致**：同輯使用相同配色、字型、排版邏輯
- **最多 10 張**：超過 7 張完讀率急降
- **最後一張放 CTA**：「儲存備用 📌」或「追蹤看更多」

| 格式 | 尺寸（px） | 備注 |
|------|-----------|------|
| 動態牆貼文 | 1200 × 630 | 1.91:1，最通用 |

| 社團封面 | 1640 × 856 | |
| 頁面封面 | 851 × 315 (桌機) / 640 × 360 (手機) | 兩端可能被裁 |

### 文字疊加限制

Facebook 廣告系統對圖片文字覆蓋率有歷史限制（舊規則為 20%，現已放寬）。但有機貼文建議：

- **有機貼文**：文字佔圖面 < 30%，保持視覺呼吸空間
- **廣告投放**：Meta 建議文字盡量少，圖片敘事為主

### 影片規格

| 項目 | 規格 |
|------|------|
| 建議尺寸 | 1280 × 720（最低）|
| 長寬比 | 16:9（橫）、4:5（直，手機效果佳）|
| 最大長度 | 240 分鐘 |
| 最大檔案 | 10 GB |
| 建議長度 | 1-3 分鐘（有機觸及）|
| 字幕 | 必要（80% 靜音播放）|

## LinkedIn

### 圖片格式與尺寸

| 格式 | 尺寸（px） | 備注 |
|------|-----------|------|
| 動態貼文（橫） | 1200 × 627 | 最通用 |
| 動態貼文（正方形） | 1080 × 1080 | 手機效果好 |
| 文章封面 | 1920 × 1080 | Article 功能 |
| 個人封面 | 1584 × 396 | |
| 公司封面 | 1128 × 191 | |

### 文件 / Carousel（Document Post）

LinkedIn 的「文件上傳」功能可作為 Carousel 替代：

- **格式**：PDF（建議）、PPT
- **頁數**：建議 5-15 頁
- **每頁尺寸**：1080 × 1080（正方形）或 1080 × 1350（4:5）
- **效果**：LinkedIn 演算法對文件貼文有機觸及顯著優於圖片
- **設計訣竅**：第一頁標題要強，最後一頁放 CTA + 追蹤提示

### 影片規格

| 項目 | 規格 |
|------|------|
| 長寬比 | 16:9、1:1、4:5、9:16 皆可 |
| 建議尺寸 | 1920 × 1080 |
| 最大長度 | 10 分鐘 |
| 最大檔案 | 5 GB |
| 建議長度 | **30-90 秒**（B2B 受眾耐心有限）|
| 字幕 | 必要 |

### LinkedIn 視覺設計守則

- 避免過度設計感強的視覺（娛樂色彩重）——**乾淨、資訊密度高** 的設計更符合平台氛圍
- 資訊圖表（infographic）在 LinkedIn 表現尤佳
- 個人頭貼清晰度直接影響信任感：建議 400 × 400 以上

## X / Twitter

### 圖片格式與尺寸

| 格式 | 尺寸（px） | 長寬比 | 備注 |
|------|-----------|--------|------|
| 單張圖片 | 1600 × 900 | 16:9 | 時間軸顯示裁切為 2:1 |
| 時間軸安全顯示 | 1200 × 675 | 16:9 | 完整顯示不裁切 |
| 正方形 | 1080 × 1080 | 1:1 | 可用但不佔最大版面 |
| 4 張並排 | 各 1200 × 675 | 16:9 | 超過 4 張不支援 |

> **注意**：X 時間軸會將圖片裁切為約 2:1 的預覽，主視覺元素應置中偏上，避免被截。

### 影片規格

| 項目 | 規格 |
|------|------|
| 最大長度 | 2 分 20 秒（免費）/ 更長（X Premium）|
| 最大檔案 | 512 MB |
| 建議長度 | **15-45 秒**（X 受眾注意力短）|
| 格式 | MP4 / MOV |
| 長寬比 | 16:9 或 1:1 |

### X 視覺使用建議

- X 是文字優先平台，視覺輔助文案而非主導
- 圖表、截圖、對話截圖類內容轉發率高
- 不要在每則貼文都加圖——**純文字 hot take** 往往比圖文貼文獲得更高互動

## 跨平台視覺製作決策框架

### 一套素材的改版流程

當同一主題需發布至多平台，不建議重新製作，而是從**主素材尺寸**改版：

```
主素材：1080 × 1350（4:5）— 適合 IG Feed + Facebook
    ↓ 裁切中央 1080 × 1080  → Facebook 正方形 / IG Carousel
    ↓ 裁切 1200 × 627        → LinkedIn 橫式
    ↓ 裁切 1600 × 900        → X/Twitter
    ↓ 加上 9:16 背景延伸     → Stories / Reels 封面
```

**Canva / Figma 工作流建議**：
1. 以 1080 × 1350 建立主畫布
2. 主視覺元素集中在中央 1080 × 1080 安全區
3. 上下延伸區域用背景色或模糊延伸填充
4. 匯出時使用「調整尺寸」功能，逐一確認各版本裁切

### 視覺選擇矩陣

| 貼文目的 | Instagram | LinkedIn | Facebook | X |
|---------|-----------|----------|---------|---|
| 品牌認知 | 高質感照片 / Reels | 資訊圖表 | 照片 + 短文 | 純文字或截圖 |
| 教育 / 教學 | Carousel（步驟圖）| 文件上傳 PDF | 影片 | 截圖 + 文字說明 |
| 活動宣傳 | Stories + Countdown | 活動貼文 | 活動功能 | 圖片 + 日期 |
| 產品展示 | Reels / 照片 | 圖片 + 數據 | 相簿 | 圖片 |
| 社群互動 | 問答 Stories | 純文字貼文 | 投票 | 純文字投票 |

## 字型與文字疊圖注意事項

### 最小可讀字級（依輸出格式）

| 平台輸出場景 | 建議最小字級 |
|------------|------------|
| IG Feed（手機閱讀）| 24pt（在 1080px 畫布上）|
| IG Stories 文字 | 32pt |
| LinkedIn 資訊圖表 | 20pt |
| Thumbnail / 封面標題 | 48pt 以上 |

### 對比度規則

文字疊圖需通過 WCAG AA 標準：
- 一般文字：對比度 ≥ 4.5:1
- 大型文字（18pt+ 粗體）：對比度 ≥ 3:1
- 工具：Stark（Figma 插件）或 WebAIM Contrast Checker

**常見失誤**：白字加淺色圖片背景，手機螢幕戶外亮度下完全看不清。解法：在文字下方加半透明深色遮罩（opacity 50-70%）。

## 品牌一致性 Checklist

每次發文前確認：

- [ ] 主色系符合品牌色盤（不超過 3 色）
- [ ] 字型與品牌字型一致
- [ ] Logo 位置固定（建議右下或左上）
- [ ] 圖片風格（攝影 / 插圖 / 扁平）前後一致
- [ ] 各平台尺寸已分別確認，無重要元素被截切
