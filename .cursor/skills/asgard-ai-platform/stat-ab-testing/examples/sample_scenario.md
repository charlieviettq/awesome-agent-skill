# Example: 結帳頁按鈕顏色對購買轉換率的影響

## Scenario

**公司**：Shopmatic（中型 SaaS 電商平台，月活躍用戶 ~180,000）

**背景**：
Shopmatic 的設計師提出將結帳頁主要 CTA 按鈕從灰色改為橘色。設計師認為橘色更有緊迫感，希望提升購買完成率。產品經理問：「我們需要多少用戶才能確認這個改動有效？我們預計跑兩週，這樣夠嗎？」

**現有數據（過去 30 天）**：
- 結帳頁日均流量：6,000 unique sessions/day
- 目前按鈕點擊→訂單完成轉換率（基準）：8.2%
- 團隊期望至少提升 15%（相對），即 8.2% → 9.4%
- 可接受錯誤：α = 0.05（雙尾），power = 0.80

---

## Analysis

### Step 1：樣本數計算（BEFORE 啟動實驗）

套用比例差異的樣本數公式：

```
p₁ = 0.082  (control)
p₂ = 0.094  (treatment，+15% relative)
MDE = 0.012 (absolute)

Z_α/2 = 1.96  (α=0.05, two-tailed)
Z_β   = 0.842 (power=0.80)

分子：(1.96 + 0.842)² × [0.082×0.918 + 0.094×0.906]
     = 7.854 × [0.07528 + 0.08516]
     = 7.854 × 0.16044
     = 1.260

分母：(0.094 - 0.082)² = 0.000144

n per group ≈ 1.260 / 0.000144 ≈ 8,750 sessions per group
```

**→ 需要每組 ~8,750 sessions，共 ~17,500 sessions。**

對照流量：6,000 sessions/day，50/50 split → 每組每日 3,000。

所需天數：8,750 / 3,000 ≈ **2.9 天（統計上）**

### Step 2：實驗時長規劃

純數字算出 3 天，但必須加入週期性校正：

- 週間 vs 週末用戶行為差異顯著（週末轉換率通常低 2-3 個百分點）
- 最少需覆蓋 **1 個完整 7 天週期**，推薦 **2 週（14 天）**

**→ 建議跑 14 天**，遠超統計所需，但能排除時序偏差。兩週後截止，不提前停止。

### Step 3：實驗設計

**假設**：
- H₀：橘色按鈕與灰色按鈕的訂單完成轉換率無差異
- H₁：橘色按鈕的轉換率 > 灰色按鈕（單側）— 但分析時用雙尾保守處理

**隨機化單位**：User ID（非 session），避免同一用戶在兩組之間切換。

**主要指標**：結帳頁→訂單完成轉換率

**護欄指標（不得下降）**：
- 平均訂單金額（AOV）— 防止橘色讓用戶衝動下單後退款
- 頁面報錯率
- 結帳頁跳出率

**停止規則**：固定期程（fixed-horizon），第 14 天結束後一次性分析，不中途查看 p 值。

### Step 4：實驗執行後結果（Day 14）

| 組別 | Sessions | 訂單數 | 轉換率 |
|------|---------|-------|-------|
| Control（灰色） | 41,820 | 3,517 | 8.41% |
| Treatment（橘色）| 41,950 | 3,912 | 9.33% |

**觀察差值**：+0.92 個百分點（相對提升 +10.9%）

**顯著性檢定（Z-test for proportions）**：

```
pooled p = (3517 + 3912) / (41820 + 41950) = 0.0887
SE = sqrt(0.0887 × 0.9113 × (1/41820 + 1/41950)) = 0.00197

Z = (0.0933 - 0.0841) / 0.00197 = 4.67

p-value ≈ 0.000003  (< 0.0001)
95% CI：[+0.53%, +1.31%]
```

**護欄指標**：
- AOV：Control $142，Treatment $139（差異 -2.1%，未達顯著）
- 頁面報錯率：兩組持平
- 跳出率：兩組持平

---

## Result

```markdown
# A/B Test Design: 結帳頁 CTA 按鈕顏色（灰 → 橘）

## Hypothesis
- H₀：兩組訂單完成轉換率無差異
- H₁：橘色按鈕轉換率高於灰色按鈕
- Primary metric：結帳頁訂單完成轉換率
- MDE：+15% relative（8.2% → 9.4%）

## Sample Size
- Baseline rate：8.2%
- Required N per group：~8,750 sessions
- Estimated duration：14 天（覆蓋 2 個完整週期）

## Results (post-test, Day 14)
| Metric | Control | Treatment | Diff | CI (95%) | p-value |
|--------|---------|-----------|------|----------|---------|
| 轉換率 | 8.41% | 9.33% | +0.92pp (+10.9%) | [+0.53%, +1.31%] | <0.0001 |
| AOV | $142 | $139 | -$3 (-2.1%) | [-$6, +$0.4] | 0.11 |

## Decision
**Ship** — 橘色按鈕達統計與實際顯著性（p<0.0001，CI 下限 +0.53pp 已超過業務門檻）。
護欄指標 AOV 未見顯著下降。建議全量上線，並在後續 30 天追蹤留存率，
確認提升非新奇效應所致。
```

> **注意：** 實際相對提升為 +10.9%，低於當初設定的 MDE +15%——但因樣本量充足，仍達顯著。這正是為什麼要事先計算樣本數：如果只跑 3 天（~18,000 sessions），同樣的效果量將會 p = 0.08，錯誤地判定為「無效」。
