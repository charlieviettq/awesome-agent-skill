# Social Media Crisis Response Playbook

社群媒體危機與傳統媒體危機最根本的差異：**速度與擴散是指數級的，而非線性的**。一則轉推可在 15 分鐘內讓一個局部事件變成全國新聞。本文件提供可操作的決策框架與步驟流程，直接與 SKILL.md 的 Golden Hour 鐵律銜接。

---

## 社群危機擴散模型

### PESO 觸發路徑

```
事件發生
   │
   ▼
Owned (品牌自有頻道) ──→ 可控，應在此率先發聲
   │
   ▼
Earned (媒體報導) ──→ 中等速度，有編輯把關
   │
   ▼
Shared (社群分享/轉發) ──→ 最快，無把關，情緒放大
   │
   ▼
Paid (廣告) ──→ 危機期間應立即暫停
```

**Iron Law 強化**：在 Shared 路徑被激活之前，品牌必須在 Owned 頻道發出聲明。Owned → Earned 的視窗通常是 **45–60 分鐘**，之後媒體就會主動找尋「品牌態度」的素材，無論你有沒有準備好。

---

## 危機等級分類（社群專用）

| 等級 | 觸發條件 | 反應時限 | 回應頻道 |
|------|---------|---------|---------|
| **L1 火苗** | 單一負面貼文，<500 互動，無媒體報導 | 4 小時內 | 直接回覆留言即可 |
| **L2 悶燒** | 話題標籤出現，1K+ 互動，記者開始詢問 | 2 小時內 | 社群聲明 + 客服私訊 |
| **L3 爆發** | Trending 話題，主流媒體報導，員工/供應商捲入 | **30 分鐘內 holding statement** | 全頻道 + 記者會 |
| **L4 複合危機** | 法律訴訟、政府介入、安全事故 | 立即 (CEO/法務) | 同上，但法務審核所有發文 |

分類方式：每 15 分鐘監測一次互動量增長率。若增長率 > 300%/小時，自動升一級。

## 30 分鐘 Holding Statement 流程

社群危機中，第一則貼文比後續的完整聲明更重要，因為它決定**初始敘事框架**。

### 步驟

```
T+0  危機警報觸發（監測工具 / 員工通報）
T+5  確認危機等級（L1–L4）
T+10 通知危機小組，指定社群發言人（通常是 Social Media Manager + PR Lead）
T+20 起草 Holding Statement（見模板）
T+25 法務快速過目（L3/L4 必要；L1/L2 可省略）
T+30 發布至主要社群頻道
```

### Holding Statement 社群版模板

Holding Statement 的目的是**佔領敘事位置**，不是解釋一切。字數控制在 **120 字以內**（Twitter/X 限制、注意力限制）。

```
我們注意到關於 [簡短描述事件] 的討論。

我們非常重視此事，目前正在積極了解情況。

我們將在 [具體時間，例如：今天下午 3 點前] 提供完整說明。

如有緊急需求，請聯繫 [客服帳號/@handle]。
```

**反例（不要這樣寫）**：
- ❌「我們對所有感到不舒服的人致歉」— 非具體道歉，被解讀為推卸
- ❌「目前事件尚不明朗，無法評論」— 等同沉默，會被填入惡意解讀
- ❌「我們始終以客戶為優先」— 企業語言，零可信度

## 平台差異應對矩陣

不同平台的危機動態不同，同一份聲明不能複製貼上。

| 平台 | 危機特性 | 回應策略 | 注意事項 |
|------|---------|---------|---------|
| **X (Twitter)** | 速度最快、媒體監測主戰場 | 先 Thread 發 Holding Statement，後發完整聲明 | 避免刪文（截圖永存），勿與攻擊者直接交戰 |
| **Facebook** | 留言區情緒累積，老用戶、家長族群 | 置頂貼文，主動回覆每則具名投訴 | 隱藏留言會被截圖放大，優先回應而非隱藏 |
| **Instagram** | 視覺危機（圖片/影片）、年輕族群 | Story 先發，Feed 後發正式聲明 | 不適合長文，導向 link in bio 或 X/官網 |
| **YouTube** | 影片危機（公關影片反效果） | 置頂留言聲明；嚴重時在頻道首頁發公告影片 | 影片說明欄是長文最佳位置 |
| **PTT / Dcard** | 台灣本地輿論形成關鍵場域 | 不直接在版上發文（官方帳號可信度低），透過媒體轉述 | 匿名文化，刪文會被存檔，勿試圖壓制 |
| **LINE 群組** | 封閉式傳播，謠言難追蹤 | 透過 LINE 官方帳號主動推送澄清訊息 | 無法直接介入，需靠 Owned 頻道的聲明被截圖傳播 |

## 社群回應決策樹

面對每一則負面留言/貼文，用以下決策樹判斷行動：

```
這則留言/貼文是否涉及：
├─ 安全風險（產品傷害、人身威脅）？
│   └─ YES → 立即私訊 + 公開確認已接觸，轉交客服/法務
│
├─ 媒體或有影響力的帳號（>1萬粉）？
│   └─ YES → PR Lead 直接接手，不交給社群客服
│
├─ 事實錯誤？
│   ├─ YES，有公開可查的資料佐證 → 公開更正，附來源，語氣中立
│   └─ YES，但屬內部資訊 → 私訊補充事實，不公開爭論
│
├─ 情緒性投訴（有名字/具體經驗）？
│   └─ YES → 公開致謝提出、私訊跟進解決
│
├─ 純粹謾罵 / 匿名惡意攻擊？
│   └─ YES → 不回應（回應等於放大），若違反社群規範則靜默處理
│
└─ 一般疑問/誤解？
    └─ 公開回覆，提供正確資訊，簡短有禮
```

**原則**：公開留言的回覆是寫給**旁觀者**看的，不是寫給發文者看的。

## 黃金小時內的社群行動清單

以下是 L3 爆發等級危機的前 60 分鐘逐分鐘指引：

```
00–05 分  觸發確認
  □ 確認事件真實性（不要對假消息做真實回應）
  □ 截圖保存原始來源（日後歸因分析用）
  □ 呼叫危機小組

05–15 分  快速情報收集
  □ 搜索 [品牌名] + 相關關鍵詞，了解討論規模
  □ 確認是否有媒體已報導或詢問
  □ 確認內部相關部門（產品/法務/客服）狀態

15–25 分  起草 Holding Statement
  □ 套用上述模板，填入具體資訊
  □ 選定承諾更新的時間點（必須能兌現）
  □ 確認回應頻道（至少 X + Facebook）

25–35 分  審核與協調
  □ PR Lead / 法務快速確認（L3 必要）
  □ 通知 CEO/高層（若 L3/L4）
  □ 準備客服接量爆增（提前告知客服團隊）

35–45 分  發布
  □ 先發 X，再同步 Facebook、Instagram
  □ 設定監測警報（每 15 分鐘追蹤一次互動量）
  □ 指定一人專責監看留言，不要沒有人盯

45–60 分  穩住
  □ 不要過度回覆（每則投訴都回容易失控）
  □ 記錄所有媒體詢問，統一由 PR Lead 回覆
  □ 啟動完整聲明撰寫流程（目標：T+4 小時內）
```

## 完整聲明的社群格式規範

完整聲明比 Holding Statement 晚發，但在社群上的格式同樣有約束。

### 長度建議

| 平台 | 建議字數 | 形式 |
|------|---------|------|
| X | 聲明主文 < 280 字元，長版用 Thread（3–5 則） | Thread 第一則必須能獨立傳播 |
| Facebook | 400–800 字 | 純文字為主，不加圖片（避免截圖去脈絡） |
| Instagram | 150 字 + 導向 Link in bio | 圖卡式聲明需過設計審核 |
| 官網新聞稿 | 無限制，完整版 | 作為所有平台的「母文件」來源 |

### 社群聲明結構（3C + 驗證事實）

對應 SKILL.md 的 3C 框架，社群版加入一個驗證步驟：

```
[Concern — 共情，1–2 句]
我們對於 [具體事件] 深感遺憾，[受影響對象] 的處境讓我們非常重視。

[Facts — 僅陳述已確認的事實，3–5 條]
以下是目前我們能確認的：
• [事實 1]
• [事實 2]
• 尚有部分細節仍在釐清中，我們不會在資訊完整前臆測。

[Commitment — 具體行動，動詞開頭]
我們正在採取以下行動：
1. [已完成的行動]
2. [進行中的行動]
3. [計劃中的行動，含時間點]

[Control — 展現掌握感]
[危機應對負責人職稱] 正在親自主導此事，並將於 [時間] 前提供下一次更新。

[Contact]
如您直接受到影響，請私訊我們或聯繫 [客服管道]。
```

## 常見社群危機類型與對應策略

### 1. 員工失言/不當言論流出

**特徵**：截圖傳播，個人行為被歸咎於品牌。

**策略**：
1. 不護短，明確切割（「此言論不代表公司立場」）
2. 說明已採取的人事行動（能說的範圍內）
3. 重申公司價值觀，但要具體，不要口號

**地雷**：不要在不了解完整脈絡前就急著道歉，有時截圖去脈絡，調查後可能立場不同。先說「我們正在了解全貌」。

---

### 2. 產品/服務瑕疵爆料

**特徵**：用戶 UGC（影片、照片）作為證據，高度可傳播。

**策略**：
1. 立即確認是否為真實產品（防止假爆料被當真）
2. 若屬實，立即啟動下架/暫停服務，**先行動，後聲明**
3. 提供受影響用戶具體補救方案（退款、替換），而非只有道歉

**地雷**：「我們的產品經過嚴格品管」在瑕疵已被拍下來的情況下，是火上加油的句子，永遠不要用。

---

### 3. 社群攻擊性活動（Cancel Campaign）

**特徵**：有組織的、持續的標籤攻擊，通常有觸發事件（過去言論被挖出、政治立場爭議）。

**策略**：
1. 評估發起者：媒體記者還是網路社群？前者需要正式回應，後者有時沈默比回應好
2. 避免「品牌殉道」：不需要在所有議題上表態，選擇性沉默是合法策略
3. 若核心支持者仍穩定，不需為了平息少數憤怒而做出大幅讓步

**地雷**：不要刪除正常的社群留言（即使是批評），這會引發「品牌在審查」的第二波攻擊。

---

### 4. 數據外洩/隱私事件

**特徵**：法律義務（個資法、GDPR）+ 用戶恐慌並存。

**策略**：
1. 法務優先確認揭露義務（72 小時通報規定等）
2. 聲明中必須說明：**哪些資料、多少用戶、何時發生、已採取什麼措施**
3. 提供受影響用戶可採取的自保步驟（更改密碼、凍結帳號）

**地雷**：避免說「少量用戶受影響」，如果數字後來更新，公信力徹底崩潰。寧可說「正在確認規模」。

## 危機後的社群分析報告框架

危機平息後 48–72 小時，應產出一份社群分析報告，作為 SKILL.md Recovery Phase 的輸入。

```
# 社群危機事後分析：[事件名稱]

## 擴散數據
- 觸發時間：
- 品牌首次回應時間：（距觸發 X 分鐘）
- 峰值互動量：（時間點 + 數量）
- 主要擴散帳號：（類型：媒體/KOL/一般用戶）
- 最終觸及人數（估算）：

## 敘事分析
- 主要批評論點（前 3 名）：
- 品牌支持者論點：
- 媒體報導角度：
- 最終定調的主流敘事：

## 回應效能
- Holding Statement 效果：（有效降溫 / 未能阻止擴散）
- 完整聲明後互動量變化：
- 未能預期的問題點：

## 改善建議
- 監測盲點（為何沒有更早發現）：
- 回應流程瓶頸：
- 聲明語言問題：
- 下次可預備的 FAQ：
```

## 絕對禁止清單（社群危機）

以下行為在任何情況下都不應出現：

| 禁止行為 | 原因 |
|---------|------|
| 刪除含有合理批評的留言 | 截圖永存，觸發「滅口」敘事 |
| 與匿名帳號公開爭論 | 放大仇恨，讓攻擊者獲得品牌背書的能見度 |
| 在危機期間繼續投放廣告 | 被截圖對比，顯得漠視危機 |
| 過度道歉（「我們非常非常抱歉」） | 顯示失控感，無助於解決問題 |
| 聲明中使用被動語態規避責任 | 「錯誤被發生了」是公關自殺語言 |
| 在不同平台發出矛盾訊息 | 被對比截圖，公信力歸零 |
| 要求用戶刪文或不要討論 | 觸發 Streisand Effect，加速擴散 |
