# Example: SaaS 續約談判 — 供應商漲價 40%

## Scenario

使用者輸入：

> 我們用的資料倉儲 SaaS（Snowflake 競爭對手，叫 DataVault）要來談年約續簽了。他們報價說因為硬體成本上漲，年費從 $120K 漲到 $168K，漲幅 40%。我覺得太貴，但我們的 data pipeline 已經深度整合進去，換平台代價很高。業務說「這是最終報價」。我該怎麼辦？

## Analysis

### Step 1：釐清 BATNA

**我方 BATNA**

| 替代方案 | 可行性 | 預估成本 |
|---------|--------|---------|
| 遷移至 BigQuery | 6-9 個月工程工期 | 遷移一次性成本 ~$80K + BigQuery 年費 ~$95K |
| 遷移至 Redshift | 同上 | 遷移 ~$70K + Redshift 年費 ~$88K |
| 維持現狀、接受 $168K | 無額外成本 | $168K/年 |

**BATNA 評分**：Redshift 方案是最佳替代。年費 $88K，但加上攤提遷移成本（3 年攤提 = $23K/年）→ **實際成本 $111K/年**。

這代表：若 DataVault 願意談到 $111K 以下，遷移才划算；但光是工程人力中斷和業務風險，$115K-120K 之間仍可接受。

**他方 BATNA**

DataVault 的替代方案是「丟掉這個客戶」：
- 你們年費貢獻 $120K，是穩定收入
- 客戶深度整合 = 高黏著，churning 罕見
- 替換一個 enterprise 客戶的取得成本（CAC）通常是年費的 1-2 倍
- **他們的 BATNA 很差** — 失去你們比幫你調整報價更痛

---

### Step 2：計算 ZOPA

| 元素 | 我方 | 他方（估計） |
|------|------|------------|
| 開價 | $120K（續約無漲價） | $168K |
| 保留點（最差能接受） | $120K（不漲價為底線） | ~$130K（低於此可能內部過不了關） |
| 期望點 | $115K（微降以反映合約調整） | $160K |
| ZOPA | **$130K–$120K** 有重疊 ✅ |

ZOPA 存在，但空間窄。談判重點在於創造價值（條款、期限、功能包），而非純粹殺價。

---

### Step 3：分離立場與利益

| | 立場（表面） | 利益（深層） |
|--|------------|------------|
| **我方** | 不要漲價 | 可預算性、避免工程遷移風險、維持團隊產能 |
| **他方** | $168K 是最終報價 | 達到 ARR 目標、維持毛利、不開壞先例給其他客戶 |

**關鍵洞察**：他們說「最終報價」是立場，背後的利益是「不能被其他客戶拿來當談判籌碼」。解法是讓降價「有理由」，而非單純讓步。

---

### Step 4：擴大選項（創意方案）

1. **延長合約換降價**：簽 3 年約，年費 $135K，DataVault 換取更長 ARR visibility
2. **預付折扣**：一次付清 2 年，要求 10% 折扣 → $302K（相當 $151K/年）
3. **功能升級換接受漲幅**：接受 $148K，但要求加入企業級 SSO + 優先支援 SLA（原本要另外付費）
4. **縮減使用量**：降回 $120K，但降 compute credits 配額 20%（強迫他們重新評估價值）
5. **引入競品報價**：取得 BigQuery 或 Redshift 的概念性報價，作為錨點

---

### Step 5：客觀準則

- SaaS 年漲幅市場慣例：3-8%（40% 遠超正常）
- DataVault 競爭對手（BigQuery enterprise）同規格報價：$105K–$115K/年
- 通膨指數（2025）：約 3.2%，不足以支撐 40% 漲幅
- 你們的使用量 YoY 是否成長？若持平，漲幅更難自圓其說

---

## Result

# Negotiation Prep: DataVault 年約續簽

## Position Analysis

| Element | 我方 | DataVault（估計） |
|---------|------|-----------------|
| BATNA | Redshift 遷移（實際成本 $111K/年含攤提） | 丟失 $120K ARR 客戶（CAC 重置 $120-240K） |
| Reservation price | $120K（不漲價底線；高於此需要換取具體價值） | ~$130K（低於此毛利目標可能違規） |
| Aspiration price | $115K + 功能升級 | $168K |
| ZOPA | **$130K–$120K**，附加條款可拓寬空間 |

## Interests（不是立場）

| 我方利益（優先序） | DataVault 利益（估計） |
|-----------------|---------------------|
| 1. 預算可控、不超 $130K | 1. 達到 Q2 ARR 擴張目標 |
| 2. 避免工程遷移中斷產能 | 2. 不對外開漲價讓步先例 |
| 3. 維持資料管線穩定性 | 3. 維持毛利率 |
| 4. 合約條款有彈性（可縮減） | 4. 留住高黏著客戶 |

## Creative Options

1. **3 年約 + $135K/年**：DataVault 獲得確定性，我方控制在預算內
2. **接受 $145K，但拿企業 SSO + 優先 SLA**（原 $18K 加購）→ 實際僅漲 $7K
3. **縮減配額方案**：維持 $120K 但 compute credits 降 15%，雙方重新談使用量基準
4. **引入 BigQuery 報價**作為反錨點，要求 DataVault justify 溢價

## Objective Criteria

- SaaS 市場年漲幅慣例：3–8%（40% 無法以通膨解釋）
- BigQuery 同規格企業報價：$105–115K/年（可取得書面報價）
- 我方使用量 YoY 成長率：若 < 10%，40% 漲幅完全無依據

## Strategy

- **開場**：表達續約意願，但明確指出 40% 漲幅在市場上站不住腳，附 BigQuery 概念報價
- **反錨**：提案 $118K + 3 年約，理由是「長期承諾換合理定價」
- **讓步順序**：
  1. 先讓期限（從 1 年簽 2 年）
  2. 再讓金額（從 $118K 到 $128K）
  3. 最後要求功能升級作為收尾條件
- **不要單邊讓步**：每次讓步都說「如果你們能在價格上調整到 X，我們可以在 Y 上配合你」
- **走人訊號**：若對方堅持 $150K 以上且無實質功能增值，啟動 Redshift 遷移評估並告知 DataVault，觀察其反應
- **「最終報價」處理**：回應「我理解這對你們內部有壓力，我們試試看能不能調整結構而不是單純改數字」——繞過立場、碰利益
