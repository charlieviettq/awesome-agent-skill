# 查證方法與多方平衡

查證是記者工作的核心。沒查證的稿件是謠言，只是包裝得像新聞。

---

## 一、查證的層次

每個事實依其重要性，需達到的查證層次不同：

| 事實類型 | 最低查證要求 |
|---------|------------|
| 人物頭銜、姓名、年齡 | 至少 1 個官方／權威來源（名片、公司網頁、學歷證明） |
| 時間、地點、事件發生 | 至少 1 個一手或多個二手 |
| 數字、統計、金額 | **原始出處**（財報、政府統計、學術論文） |
| 言論、引言 | 錄音、錄影、書面；若為二手需再次與當事人確認 |
| 對他人的指控 | **至少 2 個獨立來源**，且被指控方有回應機會 |
| 歷史／過往事件 | 可信文獻、多方印證 |
| 法律條文、判決 | 原文（全國法規資料庫、司法院判決書查詢） |

## 二、五步查證法

### Step 1 — 列出所有需查證項目

初稿完成後，把**每一個具體事實**列出來：

```
待查證清單：
- [ ] 陳先生 58 歲 → 身分證字號或訪談確認
- [ ] 台積電 Q1 營收 8,392 億 → 財報原文
- [ ] 「自殺率上升 23%」→ 衛福部哪份報告？
- [ ] 「全台 12 家醫院超標」→ 哪 12 家？資料來源？
- [ ] 「主管當時發怒」→ 誰說的？錄音？
```

### Step 2 — 找原始來源

**原始來源優先順序**：

1. 一手文件（公文、財報、判決書、學術論文原文）
2. 當事人親述（錄音、錄影、文字訪談）
3. 目擊者親述
4. 其他媒體報導（**僅作參考，不作直接引用**）
5. 網路傳言（**不可作為來源**）

### Step 3 — 交叉比對

- 兩個來源說法一致？→ 可用
- 兩個來源說法不一致？→ 找第三方，或併陳兩說
- 只有一個來源？→ 在文中標示「此說未獲其他來源證實」

### Step 4 — 反向驗證

對關鍵事實做**反向驗證**：

- 假設這個說法是假的，我會看到什麼矛盾的證據？
- 主動去找那些矛盾證據
- 若找不到反證，信心度提高；若找到反證，重新評估

### Step 5 — 給被指控方回應

涉及對他方的負面指控時：

- 正式發函／發信（留紀錄）
- 給合理回應時間（通常 3-7 天，依緊急程度）
- 若口頭回應，事後書面確認
- 聯繫未果需具體紀錄方式：「於 4/15 下午 3:22 撥打 02-XXXX-XXXX，無人接聽」

## 三、查證工具

### 3.1 台灣常用公開資料

| 主題 | 工具 |
|------|------|
| 公司資料 | 經濟部商業司商工登記資料 |
| 上市櫃財報 | 公開資訊觀測站 mops.twse.com.tw |
| 法院判決 | 司法院裁判書查詢 |
| 法規條文 | 全國法規資料庫 |
| 政府公開資料 | data.gov.tw |
| 政府採購 | 政府電子採購網 |
| 土地／不動產 | 地政電子謄本、實價登錄 |
| 學校系所 | 各校系所官網 |
| 新聞查核 | 台灣事實查核中心 tfc-taiwan.org.tw、MyGoPen、Cofacts |

### 3.2 國際公開資料

| 主題 | 工具 |
|------|------|
| 國際公司 | OpenCorporates |
| 學術論文 | Google Scholar、PubMed、PLOS |
| 衛星影像 | Google Earth、Planet Labs |
| 國際法庭 | ICC、ICJ 官網 |
| 跨國資料新聞 | ICIJ、Bellingcat 方法論 |

### 3.3 圖片／影片查證

- **反向圖片搜尋**：Google Images、TinEye、Yandex
- **Metadata 檢查**：ExifTool（看拍攝時間、地點、設備）
- **AI 生成檢測**：目前沒有 100% 可靠的工具；用多個檢測器 + 人工判斷
- **地理定位**：Google Maps 街景比對、影片中可見標誌物

### 3.4 社群媒體查證

- 帳號歷史（創建時間、過往發言、粉絲互動真實度）
- 同一訊息在多少獨立帳號出現？是否有機器人傳播特徵？
- 原始發文者是否可聯繫？

## 四、多方平衡的實踐

### 4.1 什麼是「平衡」

**不是**：甲方說 A，乙方說 B，各留一段，不做判斷——這是「虛假平衡」（false balance）。

**是**：

- 涉及**爭議**的議題，呈現各方論點
- 涉及**科學共識**的議題（如氣候變遷），不給異議者同等版面（會誤導讀者）
- 涉及**事實查核**的議題，指出哪方說法與證據相符

### 4.2 平衡的對象

涉及以下主體時，都要考慮是否給予回應機會：

- 被指控方（個人、公司、機關）
- 受影響方（員工、居民、消費者）
- 主管機關
- 專業社群（醫界、學界、法界）
- 利益相關方（股東、供應商、客戶）

### 4.3 當對方拒絕回應

處理方式（按優先順序）：

1. **記錄聯繫過程**：時間、方式、回應
2. **文中明載**：「本刊於 X 日以電話、電郵聯繫 Y，截至發稿前未獲回應」
3. **善意詮釋**：不假設對方有惡意，僅陳述事實
4. **持續嘗試**：若稿件可延發，繼續聯繫；事件持續中可後續更新

### 4.4 當對方威脅／施壓

- 記錄所有威脅內容（錄音、截圖）
- 通報主管、法務
- **不**讓威脅影響稿件真實性（事實就是事實）
- **但**可重新評估來源保護、發稿時機

## 五、查證常見陷阱

### 5.1 二手資料當一手用

「根據某媒體報導」≠ 查證。必須回到原始來源。

### 5.2 PDF 當原始文件

PDF 可偽造。取得 PDF 後應：

- 檢查 Metadata（作者、修改時間）
- 從官方網站／原始機構二次確認
- 與其他公開紀錄比對

### 5.3 錄音逐字稿不回看

訪談錄音應**自己**（或經手人）至少聽過一次，確認逐字稿無誤。AI 轉錄需人工校對。

### 5.4 「大家都知道」的事實

有些「人人皆知」的事實實則只是都市傳說。例：「愛因斯坦數學成績很差」→ 錯。查證時**不要假設**。

### 5.5 專家的權威幻覺

引用的專家也可能錯。可能原因：

- 該專家專業不在此議題（醫師評論法律）
- 該專家有利益衝突
- 該專家說法已過時（引用 10 年前論文）
- 該專家的意見不代表學界共識

盡可能引用**多位**專家、說明其背景、查證其發言是否符合學術共識。

### 5.6 「我當時也在現場」

記者親歷不等於準確。記憶有選擇性、觀察有盲點。親歷亦須書面紀錄 + 他人佐證。

---

## 六、查證的極限

有些事實**無法 100% 查證**：

- 當事人內心動機
- 未被記錄的私下對話
- 時間久遠、證人離世、資料銷毀的歷史事件

處理原則：

- **承認不確定**：「動機不明」「細節難以還原」
- **呈現多種可能**：列出幾種解讀與證據強度
- **不強行下結論**：讓讀者自行判斷

---

## 七、查證的時間成本

查證需要時間——這是記者工作的**投資**，不是「拖延」。

| 稿件類型 | 合理查證時間比例（查證 / 撰寫） |
|---------|---------------------------|
| 即時新聞 | 1:1（半小時查證、半小時寫） |
| 一般新聞 | 2:1 |
| 深度報導 | 5:1 至 10:1 |
| 調查報導 | 10:1 至 20:1 |

**查證時間短於撰寫時間 → 警訊**。寧可延後發稿，不要發不實報導。
