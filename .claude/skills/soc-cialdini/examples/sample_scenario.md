# Example: 台灣 B2C SaaS 訂閱轉換率優化

## Scenario

**公司背景：** Flowdesk，一家台灣本土的電商賣家後台工具（SKU 管理、訂單同步、報表），月費 NT$499，目前有 1,200 付費用戶。

**用戶的問題：**
> 我們的免費試用（14 天）轉換率只有 8%，業界平均是 15-20%。試用期結束時寄出的升級信開啟率 43%，但點擊付款的只有 8%。Landing page 也覺得很空洞，但不知道從哪裡改起。請幫我分析哪裡出問題，以及怎麼改。

**已知資料：**
- 試用用戶月增約 300 人，轉換 ~24 人
- 現有 Landing page 文案：「強大的電商後台工具，讓您的業務更有效率」
- 目前沒有任何用戶評價、媒體曝光或認證標章
- 試用期結束信只有一封，主旨：「您的試用即將到期」
- 定價頁只有方案比較表，無其他元素

---

## Analysis

### Step 1 — 識別說服情境

- **說服方：** Flowdesk 產品團隊
- **受眾：** 已試用 14 天的台灣中小型電商賣家（通常是老闆或獨立接案者，決策者即付款者）
- **目標行動：** 輸入信用卡、完成首月訂閱
- **目前轉換率：** 8%（目標：15%）

台灣電商賣家屬於**集體主義文化 + 高度風險規避**，對以下原則反應最強：Social Proof（同業都用）、Authority（官方認可）、Reciprocity（先得到再付出）。

---

### Step 2 — Principle Audit（現有狀態）

| Principle | Currently Used? | How | Effectiveness |
|-----------|----------------|-----|---------------|
| Reciprocity | 部分 | 14 天免費試用，但試用期間無額外給予 | L — 給了時間，沒給額外價值 |
| Commitment | N | 無任何引導完成設定或小目標的機制 | — |
| Social Proof | N | 無用戶數、評價、案例 | — |
| Liking | N | 文案生硬、無品牌個性、無賣家社群感 | — |
| Authority | N | 無任何認證、媒體、合作夥伴標章 | — |
| Scarcity | N | 試用到期信未製造任何緊迫感 | — |

**診斷：** 六個原則只被動用了 Reciprocity 的最基本形式（免費試用本身），其餘五個原則完全缺席。轉換率 8% 純靠產品力撐起，有很大提升空間。

---

### Step 3 — 具體建議

#### 建議 1｜Reciprocity — 在試用期間加碼給予

**現狀問題：** 14 天試用只是「開門讓你進來看看」，沒有主動交付額外價值。

**實作：**
- 試用第 3 天自動寄送「電商 SKU 健診報告」（用他們已上傳的資料自動生成）
- 試用第 7 天提供「高轉換商品頁文案模板包」（10 份，可直接套用）
- 這些東西免費、有用、不需要付費就能帶走

**預期效果：** 賣家心理上欠了一份情，升級的摩擦降低。

**倫理檢核：** 報告和模板對賣家是真實有用的工具，不是誘餌。✓

---

#### 建議 2｜Commitment & Consistency — 試用第 1 天的「設定引導」

**現狀問題：** 用戶進來後沒有明確的成功路徑，不知道「試用成功」長什麼樣子。

**實作：**
- 註冊後立即顯示 3 步驟快速設定向導：①匯入商品 ②連接蝦皮/Shopee ③查看首張報表
- 完成後顯示「✓ 您已完成 Flowdesk 基本設定，NT$499/月 解鎖完整功能」
- Onboarding 過程中詢問：「您希望每月節省多少時間在訂單管理上？」→ 用戶說出數字後，轉換信可以回應：「您說希望節省 5 小時/月 — 這是我們幫您計算的結果…」

**預期效果：** 用戶對「我是 Flowdesk 用戶」這個身份產生了小承諾，更容易繼續。

**倫理檢核：** 引導設定是真實幫助用戶上手，不是操控。✓

---

#### 建議 3｜Social Proof — Landing page + 定價頁注入具體數字

**現狀問題：** 完全沒有社會認同訊號，賣家無法判斷「是否有人跟我一樣在用」。

**實作（Landing page）：**

```
現有文案（改前）：「強大的電商後台工具，讓您的業務更有效率」

改後：
1,247 位台灣電商賣家正在使用 Flowdesk 管理他們的店鋪
── 包含蝦皮月營業額 NT$50 萬以上的賣家

[圓形頭像] [圓形頭像] [圓形頭像] +1,244 位賣家
★★★★★ 平均 4.8 分（來自 312 則評價）
```

**定價頁加入：**

```
🏆 最受歡迎方案  ← 標籤加在 NT$499 方案上
「這個月新加入的 87 位賣家都選擇了這個方案」
```

**具體評價（真實用戶，加照片 + 店名）：**
> 「以前每天花 2 小時對訂單，現在 20 分鐘搞定。」— 台中賣家，每月出貨 800 單

**倫理檢核：** 所有數字必須是真實的（1,247 = 真實用戶數，4.8 = 真實評分）。✓

---

#### 建議 4｜Authority — 加入台灣電商生態認證

**現狀問題：** 新賣家不知道 Flowdesk 是否值得信任。

**實作：**
- 展示「蝦皮開放平台認證合作夥伴」標章（如已是 API 合作者）
- 「資策會電商加速器」或類似加速器校友身份
- 媒體露出：「數位時代」、「Inside」有報導過則加 "As seen in" logo 列

**倫理檢核：** 只展示真實擁有的認證和真實的媒體報導。✓

---

#### 建議 5｜Scarcity — 試用到期信重寫（限時 + 損失框架）

**現狀問題：** 主旨「您的試用即將到期」是中性通知，沒有動機驅動。

**改寫試用結束信（共 2 封）：**

第 1 封（到期前 48 小時）：
```
主旨：您還有 2 天 — 這份資料可能白花了

[姓名]，

您在試用期間已建立了 {X} 個 SKU、同步了 {Y} 筆訂單。
這些資料在試用結束後 72 小時將會被清除。

如果您在 4 月 12 日前（本週五）升級，
我們幫您保留所有資料，並提供首月 8 折優惠：NT$399。

[立即升級，保留我的資料]
```

第 2 封（到期當天）：
```
主旨：最後一天：NT$399 優惠 — 明天恢復原價

今天是您的試用最後一天。
8 折優惠在今晚 23:59 結束。

還有 {X} 小時  ← 動態倒數顯示
```

**倫理檢核：** 資料清除是真實的系統行為（非虛假威脅），8 折期限是真實的（到期後確實恢復 NT$499），倒數計時是真實的截止時間。✓

---

### Step 4 — 倫理總覽

所有建議均通過「感激 vs 欺騙？」測試：報告有用、折扣是真的、社會數字是真的、認證為真實擁有。

**唯一風險點：** 若資料清除未真實執行 → 立即變操控，必須確保系統層面落地。

```markdown
| Reciprocity  | 部分            | 14 天試用    | L             |
| Commitment   | N               | —           | —             |
| Social Proof | N               | —           | —             |
| Liking       | N               | —           | —             |
| Authority    | N               | —           | —             |
| Scarcity     | N               | —           | —             |

## Recommendations
1. **Reciprocity**: 試用第 3 天自動 SKU 健診報告 + 第 7 天文案模板包
   — 降低心理欠債感，提高升級意願
2. **Commitment**: 3 步驟 onboarding 引導 + 詢問省時目標
   — 建立早期身份認同，增加一致性動機

4. **Authority**: 展示蝦皮合作夥伴標章 + 媒體露出 logo 列
   — 信任建立，對高風險規避的賣家效果顯著
5. **Scarcity**: 到期前 48h + 0h 兩封信，真實資料清除威脅 + 首月 8 折
   — 推動猶豫用戶採取行動

## Ethics Check
✓ 報告與模板真實有用（非誘餌）
✓ 社會數字與評分為真實數據
✓ 認證為真實擁有，媒體報導為真實露出
✓ 8 折優惠期限真實執行，倒數非虛構
⚠ 風險：資料清除威脅必須在系統層真實執行，否則違反 Iron Law
```

**預估影響：** 五項建議同時實作，預估轉換率可從 8% 提升至 13–16%，每月新增 15–24 名付費用戶（MRR 增加 NT$7,500–12,000）。

**優先順序（成本由低到高）：**
1. Social Proof — 只需改文案，零工程成本
2. Scarcity — 到期信改寫，輕量 email 觸發
3. Commitment — Onboarding 向導，需前端工程
4. Reciprocity — 自動報告生成，需後端資料管道
