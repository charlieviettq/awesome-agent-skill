# Example: 半導體封裝廠錫球高度製程管制

## Scenario

台中某半導體封裝廠（Packtech Co.）生產 BGA 封裝元件，品管工程師 Emily 負責監控錫球高度（Ball Height）。規格要求 300 ± 25 µm（即 275–325 µm）。

近期客戶抱怨部分批次焊接良率下降，Emily 懷疑製程有漂移。她收集了過去 6 週共 30 個抽樣子群（每群 5 個量測值，來自同一機台、同一班別），委託 Asgard 分析：「請幫我建立 X-bar/R 管制圖，判斷製程是否在管制狀態。」

## Analysis

### Phase 1: Input Validation

- 子群數量：30 ✓（≥25 門檻）
- 子群大小：5 ✓（標準 X-bar/R 適用範圍）
- 時間順序：依生產週序排列 ✓
- 量測系統：前次 Gauge R&R 報告顯示 %R&R = 7.3% ✓（< 10% 門檻）
- **Gate 通過**：可進行 Phase 2

---

### Phase 2: Core Algorithm

**基本統計量：**

```
grand mean (X̄̄) = (300.2 + 299.8 + ... + 309.0) / 30
               = 9,093.6 / 30
               = 303.12 µm

average range (R̄) = (8 + 6 + ... + 12) / 30
                  = 296 / 30
                  = 9.87 µm
```

**查表：n=5 的管制圖常數**

| 常數 | n=5 值 |
|------|--------|
| A₂   | 0.577  |
| D₃   | 0 (LCL_R 取 0) |
| D₄   | 2.114  |

**X-bar 管制圖界限：**

```
UCL_X̄ = X̄̄ + A₂ × R̄ = 303.12 + 0.577 × 9.87 = 303.12 + 5.70 = 308.82 µm
CL_X̄  = 303.12 µm
LCL_X̄ = X̄̄ - A₂ × R̄ = 303.12 - 5.70 = 297.42 µm
```

**R 管制圖界限：**

```
UCL_R = D₄ × R̄ = 2.114 × 9.87 = 20.86 µm
CL_R  = 9.87 µm
LCL_R = D₃ × R̄ = 0 (n=5 時 D₃=0)
```

---

### Phase 3: Verification — Western Electric 判讀

**X-bar 圖逐點檢核：**

| 規則 | 觸發點 | 說明 |
|------|--------|------|
| 點超出管制界限 (±3σ) | 子群 29 (308.6)、子群 30 (309.0) | 超出 UCL_X̄ = 308.82 |
| 連續 7 點遞增趨勢 | 子群 14–22（X̄ 從 301.8 持續上升至 306.0） | 8 點單調遞增，觸發趨勢規則 |
| 連續 7 點同側 | 子群 16–30（所有點均在中心線上方） | 15 點連續偏高，觸發偏移規則 |

**R 圖：**

- 所有全距均在 [6, 14] µm，遠低於 UCL_R = 20.86 µm
- R 圖在管制內，表示組內變異穩定 → 問題是系統性均值漂移，非隨機散佈增加

**IRON LAW 確認：**

雖然子群 1–20 的均值仍在規格內（275–325 µm），但製程在第 14 子群起已進入失控狀態。管制界限（±3σ from process mean）≠ 規格界限。必須先處理失控問題，再評估製程能力。

**工程師行動建議：**

1. **立即停機確認**（子群 29–30 已超出 UCL）：檢查錫膏印刷機的刮刀壓力或錫膏厚度設定是否自第 7 週起異常
2. **追溯子群 14 前後的製程紀錄**：換班交接、模具清洗、錫膏批號更換是最常見的漸變起因
3. **R 圖穩定是好訊息**：組內變異未擴大，代表量測系統正常，且漂移原因很可能是單一系統性因素（非隨機）
4. **修正後重新建立基準**：排除異常批次後，以新的穩定子群重新計算管制界限，勿沿用本次含漂移資料的界限
```

幾個設計決策說明：

- **資料設計**：子群 1–13 刻意設計為穩定期（均值在 299–302 µm），子群 14 起呈持續上漂，子群 29–30 突破 UCL——同時觸發三條 Western Electric 規則，展示多訊號共存的真實情況
- **R 圖故意穩定**：強調「均值漂移 ≠ 散佈增加」，這是 X-bar/R 圖最常見的誤解
- **IRON LAW 段落**：明確說明製程在子群 14 就已失控，但那時仍在規格內，呼應「管制界限 ≠ 規格界限」的核心鐵律
- **行動建議**：從 R 圖穩定這個線索推導出「單一系統性原因」，示範如何用 SPC 結果縮小調查範圍
