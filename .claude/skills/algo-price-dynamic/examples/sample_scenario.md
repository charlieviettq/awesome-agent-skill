# Example: 演唱會門票最後 72 小時動態定價

## Scenario

**TicketFlow** 是一家台灣票務平台，代理一場在台北小巨蛋舉辦的獨立樂團演唱會（座位數：13,500）。

活動日期：2026-04-18（週六）
現在時間：2026-04-15 10:00（距演出 72 小時）

產品經理 Cathy 帶著以下資料來詢問：

> 「我們現在賣了 11,200 張票，剩 2,300 張。這週搜尋量突然衝高，競品平台的黃牛票掛價都到 2,800 元。我們官方售價還在 1,800 元。要怎麼調？會不會調太高引爆客訴？」

平台當前資料：
- 官方售價：NT$1,800（普通區統一定價）
- 已售：11,200 / 13,500（83% 佔用率）
- 剩餘庫存：2,300 張
- 近 24 小時搜尋量：+340%（vs 過去 7 日平均）
- 近 24 小時購票轉換率：18%（平時 6%）
- 競品黃牛平台均價：NT$2,800
- 價格上限（主辦授權）：NT$2,500
- 價格下限（合約保護）：NT$1,800

---

## Analysis

### Phase 1: 輸入驗證

| 資料流 | 狀態 | 說明 |
|--------|------|------|
| 需求訊號 | ✅ 即時 | 搜尋量 +340%，轉換率 3× 基準 |
| 庫存狀態 | ✅ 即時 | 剩餘 17%（2,300 張），屬高稀缺區間 |
| 競品價格 | ✅ 可用 | 黃牛均價 NT$2,800（非官方競品，參考用） |
| 業務規則 | ✅ 確認 | Floor NT$1,800 / Ceiling NT$2,500 |

**Gate 通過**：三條資料流均為當日資料，業務規則已界定邊界。

---

### Phase 2: 核心演算法 — 需求曲線法

**步驟 1：估算當前需求彈性**

從平台歷史數據取得同類演出的價格-銷量曲線。保守估算：

| 假設售價 | 預估剩餘 72H 轉換率 | 預估增售張數 |
|----------|---------------------|-------------|
| NT$1,800（不動）| 18% | 414 張 |
| NT$2,000 | 14% | 322 張 |
| NT$2,200 | 10% | 230 張 |
| NT$2,500（上限）| 6% | 138 張 |

> 轉換率下滑假設來自彈性係數 ε ≈ −0.8（非必需娛樂品，短時間窗口，替代選項少）

**步驟 2：計算各價位預期收益**

```
R = 售價 × 預估增售張數

R(1,800) = 1,800 × 414 = 744,720
R(2,000) = 2,000 × 322 = 644,000
R(2,200) = 2,200 × 230 = 506,000
R(2,500) = 2,500 × 138 = 345,000
```

> 以純收益最大化，NT$1,800 仍最高——**但這忽略了稀缺性溢價與庫存耗盡風險**。

**步驟 3：加入庫存約束調整**

剩餘 17% 庫存（< 20% 閾值）觸發稀缺性加成。規則：
- 庫存 15–20%：+10% 基礎加成
- 距活動 < 96H：+5% 時間加成
- 需求訊號高燒（搜尋 +300%+）：+5% 需求加成

加成合計：+20% → 建議在 NT$1,800 基礎上調整至 **NT$2,160**，取整至 **NT$2,200**。

**步驟 4：公平性與客訴風險評估**

- 採「時間分段」定價（非個人差別定價）：公開公告「最後 72 小時票價調整」
- NT$2,200 vs 黃牛 NT$2,800：官方仍有 21% 折讓，提供購買正當性
- 不追蹤個別用戶調價，避免個人化差別定價引發 GDPR/個資疑慮
- **反製造商訓練效應**：明確聲明「活動前不降價」，阻止觀望行為

**步驟 5：競品價格戰防護**

黃牛屬非正式競品，不跟隨其定價邏輯。絕對不自動比照 NT$2,800——
官方票的信任溢價（正版保障、退換票保護）本身即差異化。

---

### Phase 3: 驗證計畫

| 指標 | 監控頻率 | 警示閾值 |
|------|----------|----------|
| 每小時新增銷量 | 每小時 | 跌破 20 張/H 觸發降價評估 |
| 客服投訴量 | 即時 | 單日 > 50 件重新評估定價策略 |
| 剩餘庫存 | 即時 | 降至 500 張以下啟動 Ceiling 評估 |
| 競品黃牛價 | 每 4 小時 | 僅作參考，不自動聯動 |

**Cathy 的操作建議：**

1. **立即調價至 NT$2,200**，在購票頁面顯示「距活動 72 小時，票價已更新」
