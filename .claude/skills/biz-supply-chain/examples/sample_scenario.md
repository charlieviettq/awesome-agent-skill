# Example: 台灣保健食品品牌供應鏈診斷

## Scenario

**公司**：NutriForce Taiwan（台灣直銷保健食品品牌，年營收 4.2 億台幣）
**管道**：官網 DTC + momo + 蝦皮，自有倉在桃園，原料供應商分布台灣、紐西蘭、日本

**用戶問題**：
> 我們最近客訴暴增，主要是兩個問題：1）常常缺貨，客人下單後等超過兩週才到貨；2）退貨率從去年 6% 爬升到今年 11%，我們不知道從哪裡下手改。

---

## Analysis

### Step 1：供應鏈地圖

```
[原料供應商] → [Source] → [Make: 代工廠/新北市] → [Deliver: 桃園倉→宅配] → [客戶]
                  ↑               ↑                          ↑
               [Plan: S&OP 每月一次，Excel 管理]
                              [Return: 客服手動處理]
```

**節點清單**：
- 原料：乳清蛋白（紐西蘭，前置時間 45 天）、膠原蛋白肽（日本，前置時間 30 天）、本地草藥萃取（台灣，7 天）
- 代工廠：1 家（台北小型代工，產能 50,000 瓶/月）
- 倉儲：自有，桃園，面積 200 坪
- 物流：黑貓宅急便（主要）＋ 蝦皮物流（蝦皮訂單）

---

### Step 2：各流程量測

| Process | 指標 | 現況 | 行業基準 | 缺口 |
|---------|------|------|----------|------|
| Plan | 預測準確率 | 54% | 85%+ | **-31%** |
| Plan | 庫存天數 | 52 天（成品）/ 78 天（原料） | 30–45 天 | 原料過多、成品過少 |
| Source | 供應商準時率 | 紐西蘭供應商 71%，日本 88%，台灣 96% | 95%+ | 紐西蘭 **-24%** |
| Source | 入料不良率 | 1.2% | <1% | 微超標 |
| Make | OEE | 未測量（僅追蹤產量） | 85%+ | 無資料 |
| Make | 計劃達成率 | 82% | 95%+ | **-13%** |
| Deliver | 完美訂單率 | 87% | 95%+ | **-8%** |
| Deliver | 訂單到交付週期 | 平均 11 天（應為 3 天） | 3–5 天 | **+8 天** |
| Return | 退貨率 | 11% | <5% | **+6%** |

**退貨原因分類**（抽樣 200 筆退貨單）：
- 產品品質問題（味道、顏色異常）：48%
- 錯誤出貨（SKU 發錯）：27%
- 貨損（包裝破損）：18%
- 其他（改變心意）：7%

---

### Step 3：最弱環節識別

**Plan 是根本問題**（預測準確率僅 54%，缺口最大）

邏輯鏈：
1. Plan 差 → 無法精準備料 → 原料庫存積壓（78 天）且錯誤品項不足
2. 原料不足時 → Make 計劃打亂（達成率 82%）→ 成品補不上
3. 成品缺貨 → 訂單積壓 → 交付週期膨脹至 11 天
4. 客戶久等 + 代工廠趕工品管鬆散 → 品質投訴 → 退貨率上升

**違反 Iron Law 的舊做法**：公司之前只針對 Deliver 改善（換更快的貨運公司），但根因在 Plan，換貨運沒用。

---

### Step 4：改善優先順序

**P1（立即）— Plan：提升預測準確率**
- 現況：只看上月銷售，沒有結合促銷行事曆、平台大促（618、雙11）
- 行動：建立滾動式 13 週需求計劃，把平台促銷檔期（已知 8 週前）納入預測模型
- 目標：3 個月內預測準確率從 54% → 72%

**P2（30 天）— Source：解決紐西蘭供應商風險**
- 現況：單一來源，前置時間 45 天，準時率 71%
- 行動：引入第二供應商（澳洲備用）；與主供應商協議 VMI（供應商管理庫存），保持 30 天安全庫存
- 目標：準時率 95%+；斷料風險降至可控

**P3（60 天）— Return：處理品質與撿貨錯誤**
- 退貨原因 48% 品質 + 27% 錯發，合計 75% 可預防
- 行動①：要求代工廠導入出貨前 AQL 抽檢（Level II），不良率目標 <0.5%
- 行動②：倉庫導入掃碼出貨驗證（每個 SKU 出貨前掃條碼比對），消除人工撿貨錯誤
- 目標：退貨率從 11% → 6% 以下（節省年逆向物流成本約 80 萬台幣）

## Result

```markdown
# Supply Chain Analysis: NutriForce Taiwan

## Supply Chain Map
紐西蘭/日本/台灣原料供應商 → [Source] → [Make: 新北代工] → [Deliver: 桃園倉→黑貓] → 客戶
                                  ↑ [Plan: 月度 Excel S&OP] ↑
                                          [Return: 客服手動] ←

## SCOR Performance Dashboard
| Process | Key Metric | Current | Benchmark | Gap |
|---------|-----------|---------|-----------|-----|
| Plan | 預測準確率 | 54% | 85%+ | **-31%** |
| Source | 供應商準時率（主要）| 71% | 95%+ | **-24%** |
| Make | 計劃達成率 | 82% | 95%+ | -13% |
| Deliver | 完美訂單率 | 87% | 95%+ | -8% |
| Return | 退貨率 | 11% | <5% | **+6%** |

## Weakest Link Analysis
**Plan**（缺口 -31%）是根因。預測不準導致備料錯誤 → 代工計劃失控 →
成品缺貨積壓 → 交付週期膨脹至 11 天。退貨率惡化則疊加了
代工品質與倉庫撿貨錯誤問題，非 Deliver 本身問題。

## Improvement Recommendations
1. **Plan**：建立滾動 13 週需求計劃 + 促銷行事曆整合
   → 預測準確率 54% → 72%；成品庫存天數從 52 天降至 35 天（釋放庫存佔用資金約 200 萬台幣）
2. **Source**：引入澳洲備用供應商 + 紐西蘭主供 VMI 協議
   → 準時率 71% → 95%；斷料風險消除
3. **Return**：代工 AQL 抽檢 + 倉庫掃碼出貨驗證
   → 退貨率 11% → 6%；年節省逆向物流成本 ~80 萬台幣

**不建議優先動作**：換物流商。交付週期慢的根因是缺貨積壓，
不是黑貓本身速度問題。換物流商不解決 Plan 問題，屬無效投入。
```
