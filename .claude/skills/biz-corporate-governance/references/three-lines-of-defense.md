# 三道防線：治理層視角

> **與 biz-erm 的分工**：本檔從「**董事會／審計委員會如何監督**三道防線」視角切入，完整的 IIA 2020 三道防線模型、風險胃納、風險熱圖、KRI 等風控實務請見 `biz-erm/references/three-lines-model.md`。

## 治理層的監督責任

三道防線是管理層內部的責任結構，**董事會與審計委員會是其監督者**，不屬於任何一道。

| 治理角色 | 對三道的責任 |
|---|---|
| 董事會 | 核准整體治理架構、任命關鍵職位（CFO／CRO／CAE） |
| 審計委員會 | 直接對接第三道（內稽）、監督第一、二道的獨立性與有效性 |
| 風險管理委員會（若設立） | 監督第二道整體有效性、風險胃納審議 |
| 薪酬委員會 | 避免薪酬結構鼓勵第一道越線（例如過度以業務獎金鼓勵冒險） |

## 審計委員會的三道防線檢核清單

審計委員會每季應就下列問題得到明確答案：

### 對第一道（業務單位）
1. 業務主管是否真正擁有風險（非推給風管處）？
2. 控制流程是否嵌入日常營運，還是獨立於業務之外？
3. 業務 KPI 與風險控制是否衝突？

### 對第二道（風管 / 法遵 / 合規）
1. 第二道是否**獨立於第一道**？（風管不應兼做業務決策）
2. 第二道的發現是否被系統性壓下？
3. CRO / 法遵長是否有獨立向董事會報告的管道？
4. 第二道的預算、人力是否足夠？

### 對第三道（內部稽核）
1. 內稽長（CAE）是否直接向審計委員會報告（而非向 CEO）？
2. 內稽是否獨立於第一、二道設計與執行？
3. 內稽發現的缺失是否有具體追蹤與改善時程？
4. 內稽對管理層的報告與對審計委員會的報告是否一致？

### 跨道協作
1. 升級機制是否健全？第二道發現問題時是否能繞過第一道直通治理層？
2. 重大事件是否有跨道應變小組？
3. 三道是否有定期協作會議？

## 治理層介入時點

### 例行
- 季度審計委員會：聽取第三道報告
- 半年度：風險長報告第二道狀況
- 年度：董事會核准整體三道防線架構

### 異常
- 重大風險事件：立即緊急會議
- 關鍵職位異動（CFO／CRO／CAE）：審計委員會審議接任者
- 監理機關重大檢查發現：專案監督改善

## 常見治理失靈模式

1. **CAE 向 CEO 報告**：內稽失去獨立性，稽核報告被過濾
2. **CRO 兼任業務主管**：第二道吞第一道，風險胃納被業務壓過
3. **第一道無責**：所有風險責任推給風管處，業務單位不承擔
4. **審計委員會流於形式**：會議提前 1 天才收資料、討論不深入
5. **第二、三道預算受制**：業務不想被查，預算審議時被縮減
6. **沒有升級機制**：第二道發現問題後無書面紀錄、無董事會通報義務

## 治理層的強化工具

- **獨立彙報線**：CAE 薪酬與任免由審計委員會核准
- **年度獨立會議**：審計委員會與 CAE 單獨會議（無管理層在場）
- **外部顧問權**：獨董可聘外部專家審查三道防線設計
- **吹哨者管道**：繞過所有三道、直通審計委員會
- **Executive Session**：董事會定期無 CEO 在場的閉門會議

## 與 ERM 的交界

- **本 skill 關切**：三道**是否被治理層有效監督**
- **biz-erm 關切**：三道內部如何**設計、運作、協作**

兩者必須搭配：治理監督無力 → 三道流於形式；三道實務不紮實 → 治理監督淪為紙上作業。
