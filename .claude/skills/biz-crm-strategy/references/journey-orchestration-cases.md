# Journey Orchestration 案例庫

## 架構要點重述

**Journey Orchestration ≠ Journey Map**

| Journey Map（旅程圖） | Journey Orchestration（旅程編排） |
|---|---|
| 靜態視覺化 | 動態執行系統 |
| 通常一張圖 | 多旅程並行 + 優先序 |
| 設計階段產出 | 營運階段持續執行 |
| 目的：挖痛點 | 目的：驅動行為 |

**Orchestration 四要素**
1. **Trigger**：何時啟動（時間／行為／事件）
2. **Content**：發送什麼（內容、折扣、人工聯繫）
3. **Channel**：透過哪個通路（EDM、SMS、App 推播、電話）
4. **Logic**：執行邏輯（優先序、頻率限制、A/B 分流）

## 零售／電商案例

### 案例 1：服飾電商新客首 30 天旅程

```
D0（註冊）   → 歡迎信 + 品牌故事 + 首購優惠券
D1          → 風格測驗（蒐集偏好）
D2          → 個人化商品推薦
D4          → 社群內容（真實用戶穿搭）
D7          → 如尚未購買：加強優惠 / 客服關懷
D10         → 如已購買：購後教學內容
D14         → 相關商品交叉推薦
D21         → 會員升等誘因
D30         → 二購折扣專屬
```

**KPI**
- D7 首購轉換率：> 15%
- D30 二購率：> 25%
- D30 取消訂閱率：< 5%

### 案例 2：購物車棄單挽回

```
放購物車 1h  → App 推播（若有 App）
放購物車 3h  → LINE 訊息或 SMS
放購物車 24h → EDM：商品提醒 + 限時優惠
放購物車 72h → 最後一波：折扣 / 庫存警告
未結帳 7 日  → 轉入一般再行銷廣告
```

**優化要點**
- 頻率限制：不超過 3 次接觸
- A/B 測試：折扣 vs. 庫存警告 vs. 社會證明
- 成本效益：挽回率 × 平均客單 > 介入成本

### 案例 3：生日月完整旅程

```
前 14 天 → 生日預告（儀式感）
前 7 天  → 生日券 500 + 專屬推薦
前 3 天  → 如尚未使用：提醒
生日當天 → 賀卡 + 小禮（免費小物）+ App 推播
後 3 天  → 如尚未使用券：最後提醒
後 14 天 → 使用效果追蹤 + NPS
```

**重點**
- 不要只發優惠券（情感面薄）
- 高 CLV 客戶增加人工電話
- 驚喜元素（如隨單贈送）效果強於純折扣

## 金融／保險案例

### 案例 4：信用卡新卡啟用旅程

```
核卡通過   → 恭喜信 + 卡片送達預告
卡片送達   → 激活教學 + 首刷獎勵說明
D7 未首刷  → 首刷提醒 + 紅利加碼
D14 未首刷 → 專員電話關懷
D30 未首刷 → 最後一次優惠 + 退卡選項
D90 活躍   → 進階權益介紹（機場接送、旅平險）
```

**金融特殊考量**
- 合規：優惠條款需符合金管會規定
- 客服時間：銀行時間限制
- 成本：信用卡獲客成本高（約 3,000–5,000 元），挽回價值極高

### 案例 5：壽險繳費提醒與升級

```
繳費前 30 天 → 溫馨預告
繳費前 7 天  → 多通路提醒（簡訊 + EDM）
繳費逾期 D1  → 首次提醒
繳費逾期 D15 → 理專主動聯繫
繳費逾期 D30 → 正式催收 + 寬限期提醒
年度健檢期   → 保單檢視 + 升級建議
家庭變化訊號 → 客製化建議（新生兒、房貸等）
```

## B2B 案例

### 案例 6：B2B SaaS 訂閱續約旅程

```
續約前 90 天 → 使用狀況健康度報告（自動生成）
續約前 60 天 → CS Manager 對接（1 對 1 會議）
續約前 30 天 → 提出續約方案（升級、平行、降級）
續約前 14 天 → 決策者高層溝通
續約前 7 天  → 如未回應：緊急介入
續約失敗     → Offboarding + 流失原因訪談
續約成功     → Expansion 討論（增席、升級模組）
```

**B2B 特殊要點**
- 決策鏈：使用者 ≠ 付費者 ≠ 決策者
- 合約週期長（年度 / 多年）
- 關係深度勝於短期優惠

### 案例 7：B2B 新客培育（ABM 風格）

```
目標帳戶名單 → LinkedIn + 郵件接觸
第一次會議   → 議程客製化、決策鏈盤點
POC 期間     → 每週使用分析報告
POC 結束     → 成效報告 + 正式提案
合約簽訂     → Onboarding 計畫（30/60/90 天）
上線 90 天   → 擴展討論
```

## 客服／體驗案例

### 案例 8：客訴後復原旅程

```
客訴發生     → 快速回應（1 小時內）
客訴處理中   → 每 24h 進度更新
解決完成     → 致歉 + 補償 + 主管關懷
後 3 天      → 滿意度調查
後 7 天      → 如滿意：邀請寫評論；如不滿意：主管再聯繫
後 30 天     → 完全不發行銷訊息（冷卻期）
後 60 天     → 回歸正常 Journey
```

**關鍵**：客訴後的客戶可能因處理得當成為「超級推薦者」，也可能成為公開黑名單。Journey 設計決定走向。

### 案例 9：退貨挽回旅程

```
退貨發起     → 快速處理（不刁難）
退款完成     → 主動電話／訊息了解原因
7 天後      → 如原因是商品問題：其他商品推薦
            如原因是服務：致歉 + 服務改善承諾
30 天後     → 如無回購：加強信號（新品、個人化優惠）
60 天後     → 判斷是否值得續留（依 CLV）
```

## 進階 Orchestration 技巧

### 頻率上限（Frequency Capping）
- 單通路每週上限（例：推播 3 次、EDM 2 次）
- 總接觸上限（不論通路每週 5 次）
- 依分群差異化（Champions 可更多，At Risk 要節制）

### 優先序解決（Priority Resolution）
當多個 Journey 同時觸發：
```
優先序 1：合約／法遵（續約提醒）
優先序 2：客訴處理（回應進度）
優先序 3：個人化推薦（購物車挽回）
優先序 4：促銷活動（週年慶）
優先序 5：內容行銷（新品 / 教學）
```

### A/B 測試矩陣
每個 Journey 至少測試：
- 標題（3 版本）
- CTA 按鈕文案（2 版本）
- 發送時間（2 時段）
- 通路（2 組合）

### 機器學習優化
**Send-time Optimization**
- 依個人過往開信時間預測最佳發送時段
- 提升開信率 10–30%

**Content Personalization**
- 依瀏覽歷史動態生成商品推薦
- 依 RFM 分群個人化優惠幅度

**Next-best-action**
- 預測此客戶下一步最可能採取的行動
- 動態決定下一個 Journey 節點

## 失敗案例與教訓

### 失敗 1：轟炸型 Journey
**症狀**：每週發 5 次，退訂率飆升
**原因**：行銷部追 KPI 不顧客戶感受
**教訓**：頻率上限是底線，退訂率要作為負面 KPI

### 失敗 2：跨通路重複
**症狀**：同一訊息，EDM、SMS、App 推播各發一次
**原因**：各通路獨立規劃，無整合
**教訓**：Journey Orchestration 的「O」就是 Orchestration（指揮協調）

### 失敗 3：過度自動化
**症狀**：客訴後自動發促銷訊息
**原因**：Trigger 邏輯未考慮「冷卻期」
**教訓**：自動化要有「情緒智能」，知道何時沉默

### 失敗 4：內容疲乏
**症狀**：Journey 上線半年後效果驟降
**原因**：內容未更新，客戶已熟悉
**教訓**：Journey 需要定期「換湯換藥」，不只調優參數

### 失敗 5：資料不整合
**症狀**：線上已購買，線下通路仍推廣同商品
**原因**：CDP 未建、資料不同步
**教訓**：技術基礎決定 Journey 品質上限
