# 客戶分層進階模型

## RFM 進階變體

### RFM+（加入行為維度）

| 維度 | 原 RFM | RFM+ 擴增 |
|---|---|---|
| R (Recency) | 最近消費距今 | 最近互動距今（含瀏覽、客服、評論） |
| F (Frequency) | 消費頻率 | 互動頻率（消費 + 非消費） |
| M (Monetary) | 消費金額 | 總價值（消費 + 推薦 + 分享 + UGC） |
| + E (Engagement) | — | 參與度：活動、社群、內容消費 |
| + C (Cost-to-Serve) | — | 服務成本：客訴、退貨、客服時數 |

**應用場景**
- 品牌俱樂部（Nike, Lululemon）— E 比 M 重要
- 訂閱制（SaaS）— F 比單次 M 重要
- 高端零售 — C 需納入，低毛利高服務成本客戶未必是 VIP

## CLV（Customer Lifetime Value）模型

### 基本公式

```
CLV = Σ (未來每期的預期現金流) × 折現因子
   ≈ 年均毛利 × 預期留存年數 − 獲客成本
```

### 進階模型

**確定型（Deterministic）**
```
CLV = M × (1/(1-r)) × 毛利率 − AC
  M：年均購買額
  r：年度留存率
  AC：Acquisition Cost
```

**機率型（Probabilistic，Pareto/NBD、BG/NBD）**
- 假設每個客戶有不同購買頻率與流失率
- 常用於非訂閱制零售
- Python 套件：`lifetimes` 可實作

**淨現值型（With Discount）**
```
CLV = Σ [(M_t × m) / (1+d)^t] − AC
  d：折現率（WACC 或 risk-free + risk premium）
```

### CLV 的陷阱

1. **過度樂觀**：假設永續留存率 90% 易高估
2. **忽略負 CLV 客戶**：低消費 + 高服務成本客戶可能是負價值
3. **類別差異**：不同品類 CLV 差距極大，混為一談失真
4. **單期計算缺陷**：應多期滾動更新，非一次定案

## 行為分群（Behavioral Segmentation）

### 常用行為變數

**購前行為**
- 搜尋關鍵字（品牌、品類、功能）
- 瀏覽深度（頁面數、停留時間）
- 比價行為（跨品牌、跨平台）
- 收藏、加入願望清單

**購中行為**
- 客單價分佈
- 品類偏好（單一 vs. 多元）
- 決策時長（瀏覽到下單）
- 購買時段與季節性

**購後行為**
- 回訪週期（第 X 天回訪率）
- 評論行為（寫評論、星等）
- 推薦行為（分享給朋友）
- 退貨／客訴率

### 分群演算法

**K-means**
- 最常用、易實作
- 需預設 K 值（常用 Elbow、Silhouette 決定）
- 對離群值敏感

**DBSCAN**
- 適合發現異常客戶（outlier）
- 不需預設群數
- 參數調整有技巧性

**Hierarchical Clustering**
- 產生樹狀結構，便於溝通
- 小規模資料較適用
- 可搭配熱圖視覺化

**RFM + K-means 組合**
- 先用 RFM 定義特徵空間
- 再用 K-means 壓縮為可行動群組
- 實務最受歡迎

## 生命週期分群

### 典型七階段

```
Prospect（潛在）→ First-time Buyer（首購）→ Repeat Buyer（回購）
    → Active（活躍）→ VIP（高價值）→ At Risk（流失預警）→ Churned（已流失）
                                              ↓
                                       Reactivation（喚回）
```

**各階段 KPI**
| 階段 | 主要指標 |
|---|---|
| Prospect | 造訪數、註冊率 |
| First-time | 首購轉換率、首購客單 |
| Repeat | 二購週期、二購率 |
| Active | 月活躍率、購買頻次 |
| VIP | VIP 升等率、VIP 流失率 |
| At Risk | 預警準確率、救回率 |
| Churned | 喚回率、喚回成本 |

### 階段轉移矩陣

```
          次期 Prospect First Repeat Active VIP At-Risk Churned
當期 Prospect    70%    25%    —     —     —      —      5%
當期 First-time   —     20%   50%   20%    —      —     10%
當期 Repeat       —      —    40%   45%    5%    10%     —
當期 Active       —      —     —    70%   15%    10%     5%
當期 VIP          —      —     —     5%   80%    10%     5%
當期 At-Risk      —      —     —    30%    5%    25%    40%
當期 Churned      —      —     —     5%    —      —     95%
```

**戰略意涵**
- 若 Active → VIP 升等率 < 10%，代表 VIP 吸引力不足
- 若 At-Risk → Churned 率 > 50%，代表挽回機制失效
- 這個矩陣應每季更新，是 CRM 戰略健康度最佳指標之一

## Survival Analysis（存活分析）

**概念**：將客戶視為「生命體」，分析其與公司關係存活時長

**常用模型**
- Kaplan-Meier 曲線（視覺化留存率隨時間變化）
- Cox 比例風險模型（分析哪些因素影響留存）
- 加速失效模型（AFT）

**EMBA 應用**
- 回答「為什麼某類客戶流失特別快」
- 識別流失早期訊號
- 量化不同留存策略的效益

## 預測性分群（Predictive Segmentation）

**churn prediction（流失預測）**
- 輸入：R、F、M、E、C、互動資料
- 輸出：未來 30/60/90 天流失機率
- 常用模型：Logistic Regression、XGBoost、Neural Network
- 搭配 Journey：流失機率 > 60% 觸發挽回介入

**upsell/cross-sell propensity（交叉銷售傾向）**
- 預測客戶對特定新產品的購買機率
- 常用：協同過濾、矩陣分解、深度學習
- 延伸：Asgard `algo-rec-*` 系列

## 實務分層範例（依產業）

### 零售 / 電商
**推薦分層**：RFM + 品類偏好 + 通路偏好
**原因**：品類與通路顯著影響經營策略

### 金融（銀行、保險）
**推薦分層**：AUM（資產規模）+ 生命週期階段 + 風險承受度
**原因**：單次交易意義小，長期資產關係重要

### B2B 製造
**推薦分層**：採購規模 + 產業別 + 關係深度 + 策略重要性
**原因**：大客戶少量，需個別化經營

### SaaS / 訂閱
**推薦分層**：使用強度（DAU/MAU）+ 功能滲透率 + 續約風險
**原因**：留存比單次消費重要，功能使用是續約先導指標

### 高端精品
**推薦分層**：Total Spend + 關係年數 + 品牌參與度 + 推薦貢獻
**原因**：情感關係 > 交易金額，VIP 經營極度客製化
