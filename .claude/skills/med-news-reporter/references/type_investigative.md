# 深度／調查報導範本

## 適用情境

- 揭露未公開的系統性問題（貪腐、職場霸凌、環境污染、醫療疏失）
- 資料新聞（大規模資料爬梳、統計分析）
- 長期追蹤（同一議題數月以上的持續報導）
- 需要多方信源交叉驗證的複雜議題

## 核心原則

1. **多方信源**：至少 3 個獨立信源才可確立一項關鍵事實
2. **原始文件**：優先使用第一手文件（公文、法院判決、會議紀錄、內部信件）
3. **反方回應**：被指控方**必須**有表態機會——接受訪問、書面回應、或明載拒絕回應
4. **時間追蹤**：用時間軸展現事件演變
5. **制度面分析**：不只是「誰做錯」，還要指出「為什麼會發生」「哪個制度失靈」

## 結構模板

```
╭──────────────────────────────╮
│ 1. 衝擊性 Lead（場景／案例）    │  ← 具體受害者或關鍵文件
├──────────────────────────────┤
│ 2. 問題陳述（Nut Graf）        │  ← 本文要揭露什麼系統性問題
├──────────────────────────────┤
│ 3. 證據鋪陳                   │  ← 文件、資料、多方訪談
├──────────────────────────────┤
│ 4. 被指控方／主管機關回應       │  ← 平衡原則，必要環節
├──────────────────────────────┤
│ 5. 結構性分析                 │  ← 為何發生、制度缺陷
├──────────────────────────────┤
│ 6. 對照或比較                 │  ← 他國做法、其他案例
├──────────────────────────────┤
│ 7. 結語與後續                 │  ← 主管機關回應、可能改變
╰──────────────────────────────╯
```

## 各段詳解

### 1. Lead（場景／案例切入）

以**具體**的人物、場景、文件切入，讓讀者有情感連結。

**範例**：
> 58 歲的陳姓護理師在台北某區醫院工作 18 年，2024 年 9 月某個夜班，她一次照顧 16 名病人。凌晨 3 點，她在護理站發現一份手寫筆記：「再撐下去我會出事。」5 天後，她在宿舍燒炭自殺。

### 2. Nut Graf（核心段落）

接在 Lead 之後，1-2 段，告訴讀者「這篇報導要談什麼系統性問題」。

**範例**：
> 陳姓護理師的案例並非孤例。《本刊》比對健保署、勞動部與 5 家醫學中心內部排班表，發現全台至少 12 家醫院的夜班護病比超過衛福部建議的 1:10 上限，其中 3 家甚至超過 1:15。這套超載的排班制度，是如何在主管機關監督下持續運作的？

### 3. 證據鋪陳

按類型組織：

- **書證**：內部郵件、排班表、醫療紀錄、財報
- **人證**：當事人、目擊者、吹哨者、同僚
- **物證／數據**：資料庫比對、公開統計、現場觀察

呈現時：

- 列出來源（「根據《本刊》取得的 2024 年 8 月排班表…」）
- 多方印證（「3 名現職護理師證實此排班方式自 2023 年起固定採用」）
- 敏感時可匿名但說明理由（「該名主管因仍在職，要求匿名」）

### 4. 被指控方回應（必要！）

- **必須**主動聯繫被指控方（醫院、公司、政府部門）
- 接受訪問 → 完整呈現回應要點
- 書面回應 → 引用關鍵段落
- 拒絕回應 → 明載（「本刊於 4 月 15 日以電話、電郵聯繫 X 醫院，截至發稿前未獲回應」）
- **不能**僅因對方不回應就省略此段

### 5. 結構性分析

回答「為何發生」：

- 法規漏洞（護病比未入法、罰則過輕）
- 執法不力（稽查頻率、人力不足）
- 利益結構（健保給付制度、醫院經營邏輯）
- 文化／慣習（加班文化、吹哨者保護不足）

引用專家、學者、主管機關前官員，**避免**自己下結論。

### 6. 對照比較

- 他國制度（日本、韓國、美國類似議題如何處理）
- 其他產業（同屬高壓工作的教師、消防員）
- 歷史對比（10 年前的相同議題有何不同）

### 7. 結語與後續

- 主管機關表態（如有）
- 當事人或家屬後續（如合適）
- 可能的改變方向（修法、司法追訴）
- **不要**用「期待未來改善」這種空洞結語

## 署名與匿名規範

### 具名消息來源（預設）

職稱 + 姓名 + 機構。首次完整、之後可簡寫。

### 匿名來源（例外）

**必備條件**（須全部滿足才可使用）：

1. 消息對當事人有具體風險（失業、訴訟、人身安全）
2. 沒有其他方式取得同樣資訊
3. 該消息經**另一獨立信源**印證
4. 編輯／主管同意（在文末註明「編輯已知悉匿名來源身分」）

**匿名揭露方式**：

```
一名熟悉內部排班流程的現職護理長（以下化名「李小姐」）表示…
```

並在稿尾或方法論段落說明匿名理由：

> 本報導採訪 7 名現職／離職護理人員，其中 3 人因擔心遭資方追究，
> 要求匿名。本刊已查證其在職紀錄與身分。

### 絕不匿名的情境

- 針對特定對象的人身攻擊性指控（必須具名或有其他證據）
- 對事實沒有獨特貢獻的「隨機意見」（路人、網友留言）

## 資料新聞（Data Journalism）補充

若報導基於資料分析：

1. **交代資料來源**：政府公開資料、申請取得、自建資料集？
2. **交代方法**：清理步驟、比對邏輯、抽樣方式
3. **交代限制**：資料時間範圍、未涵蓋項目、可能偏誤
4. **附上視覺化**：關鍵趨勢用圖表呈現（折線、長條、熱點圖）
5. **提供原始資料**（若無敏感性）：讓讀者可自行驗證

## 倫理紅線（深度／調查專屬）

- **不偽裝身分**採訪（除非揭露重大公共利益且無其他方式，且須編輯層決議）
- **不誘導**當事人陳述違背本意
- **不公開**可能使消息來源身分曝光的細節（如：「唯一一位女性主管」）
- **不使用**非法取得的資料（除非公共利益壓倒隱私權且有法律諮詢）
- **未成年人**一律模糊處理（代號、不露臉、不具名）
