# FSC Licensing Application Guide

金融監督管理委員會 (FSC) 核照流程的具體指引。本文件聚焦於**申請程序、文件清單、資本要求、常見駁回原因**，不重複 SKILL.md 的架構總覽。

---

## 核心原則：LICENSING FIRST

```
IRON LAW: 未取得執照即營業屬刑事犯罪。
電子支付機構管理條例第 44 條：處 3 年以下有期徒刑或拘役，
並得科新臺幣 1,000 萬元以下罰金。
```

## 許可類型速查

| 許可類型 | 主管法規 | 最低實收資本額 | 主辦單位 | 標準審核時程 |
|---------|---------|------------|--------|------------|
| 電子支付機構許可 | 電子支付機構管理條例 | NT$5 億 | 金管會銀行局 | 12–18 個月 |
| 第三方支付服務業登錄 | 第三方支付服務業管理辦法 | NT$500 萬 | 金管會銀行局 | 1–3 個月 |
| VASP 登錄 | 虛擬資產服務提供者管理辦法 | 未明定（實務上建議 NT$1,000 萬以上） | 金管會證期局 | 審查中（2024 起） |
| 投資顧問執照 | 證券投資信託及顧問法 | NT$2,000 萬 | 金管會證期局 | 6–12 個月 |
| 群眾募資平台（股權型） | 證券商設置標準 | NT$1 億 | 金管會證期局 | 9–12 個月 |

## 電子支付機構許可：完整申請流程

### Step 0：Pre-Application（3–6 個月）

在正式送件前，FSC 銀行局提供**預先諮詢**（Pre-filing consultation）。

```
諮詢方式：
1. 以書面向金管會銀行局第三組提出諮詢申請
2. 說明事項：
   a. 擬提供之電子支付服務範圍
   b. 業務流程概述（資金流向圖）
   c. 股東結構與出資來源
3. 諮詢結果不具拘束力，但可確認：
   - 是否落入電子支付定義
   - 是否需電子支付許可或僅需第三方支付登錄
   - 監理重點提示
```

**決策點：你的 App 需要「儲值」嗎？**

```
用戶能否在你的系統內預存金額，之後再消費或轉帳？
├── YES → 電子支付機構許可（NT$5億資本，12-18個月）
└── NO  → 第三方支付登錄（NT$500萬資本，1-3個月）

「儲值」定義（電子支付機構管理條例第3條）：
以電子設備儲存、移轉具有貨幣價值的資料，
供消費者用於支付或轉帳。
```

### Step 1：籌設申請（核准籌設，約 3–6 個月）

**須備文件：**

```
A. 公司基本文件
   □ 發起人名冊（含居住地、職業、持股比例）
   □ 發起人及董事無不良信用紀錄聲明書
   □ 擬設定公司章程草案
   □ 股東會議事錄（如已成立）

B. 業務計畫書（最關鍵文件）
   □ 服務範圍及業務模式說明
   □ 資金流向圖（含清算、結算架構）
   □ 風險管理機制
   □ 消費者保護措施
   □ 資訊系統安全規劃
   □ 未來三年財務預測（含損益預測、現金流量表）

C. 資本規劃
   □ 股東出資能力證明（近三年財報或財力證明）
   □ 資金來源合法性聲明
   □ 實收資本額達 NT$5 億的時程規劃
```

**業務計畫書的常見缺失（FSC 審查最常退件原因）：**

```
❌ 資金流向圖不清楚
   → 必須顯示：消費者 → 你的系統 → 商家/收款方，
     每個節點的清算時間與帳戶存放機構

❌ 風險管理流於形式
   → 必須具體說明：交易監控閾值、STR 觸發條件、
     資安事件應變程序（含通報 FSC 時限）

❌ 財務預測過於樂觀
   → 需有合理假設基礎，FSC 會比對同業數據

❌ 消費者保護措施不足
   → 必須說明：糾紛處理流程、備付款項信託安排（或銀行保全）
```

### Step 2：正式設立登記（1–2 個月）

籌設核准後：

```
1. 完成公司法人登記（經濟部商業司）
2. 繳足 NT$5 億實收資本額
3. 取得銀行資本額查核報告書
4. 完成 AML/CFT 內部規範制定
5. 完成資訊系統建置（含 DR 備援）
```

### Step 3：正式許可申請（核准許可，約 6–12 個月）

**追加文件（於籌設文件之上）：**

```
□ 公司登記證明文件
□ 董事、監察人名冊及符合資格聲明
□ 負責人適格性資料（學經歷、無犯罪紀錄）
□ 資訊系統安全審查報告（須委外進行滲透測試）
□ AML/CFT 法遵程序書（含 KYC/CDD 作業規範）
□ 消費者保護實施辦法
□ 備付款項管理辦法（信託契約或保全機制）
□ 業務委外管理辦法（如有）
```

**備付款項保全機制（必選一）：**

```
Option A：信託
└── 與信託業者簽訂信託契約
└── 備付款項全額信託給受託銀行
└── 費用：信託管理費約 0.02–0.05% p.a.

Option B：銀行保全
└── 由銀行出具保全承諾書
└── 備付款項存放於銀行獨立帳戶
└── 費用：依與銀行協議

FSC 立場：Option A（信託）較被接受，退件風險較低。
```

### Step 4：正式開業後持續義務

```
定期申報：
├── 月報：次月 15 日前，業務統計數據
├── 季報：次季第一個月末，財務報表
└── 年報：次年 4 月 30 日前，稽核報告

事前申請變更：
├── 重大業務變更（新增服務項目）→ 須事先申請
├── 董事/監察人變更 → 須報准
└── 資本額調整 → 須申請核准
```

## 第三方支付服務業登錄：完整流程

### 適用範圍確認

```
第三方支付 = 代理收付款項服務，且：
✓ 不持有消費者資金超過 T+2 個工作天
✓ 無儲值功能
✓ 不提供 P2P 轉帳
✓ 每筆交易上限：NT$3 萬（個人）/ NT$10 萬（企業）
```

### 登錄文件清單

```
□ 公司登記文件
□ 代表人身分證明
□ 業務說明書（服務範圍、資金流向）
□ 客戶資金隔離機制說明（獨立帳戶規劃）
□ 消費者申訴處理程序
□ AML 聲明書（簡化版）
□ 資訊安全基本措施說明
```

### 第三方支付 vs 電子支付：決策樹

```
你的服務是否涉及以下任一項？
│
├── 用戶可儲值（預存金額）？           → 電子支付許可
├── 用戶可 P2P 轉帳？                  → 電子支付許可
├── 單筆超過 NT$3 萬（個人）？         → 電子支付許可
└── 以上皆否                           → 第三方支付登錄

注意：若從第三方支付「升級」至電子支付功能，
      必須先取得電子支付許可，才能上線新功能。
```

## VASP 登錄（2024+ 規範）

### 登錄條件

2024 年 FSC 發布「虛擬資產服務提供者管理辦法」，規定以下業者須登錄：

```
須登錄之 VASP 服務：
├── 虛擬資產交易所（買賣撮合）
├── 虛擬資產保管業務
├── 虛擬資產轉帳服務（非個人自用）
└── 虛擬資產衍生品交易（可能需額外期貨許可）

豁免情形（暫行規定）：
├── 純技術服務提供者（不直接持有客戶資產）
└── 境外 VASP（但禁止向台灣用戶主動行銷）
```

### VASP 登錄核心文件

```
A. AML/CFT 專項文件（重中之重）
   □ 洗錢防制及打擊資恐作業規範
   □ 客戶盡職調查（CDD）程序
   □ 虛擬資產轉帳 Travel Rule 合規方案
       （依 FATF Recommendation 16）
   □ 交易監控系統說明（鏈上分析工具）
   □ STR 申報程序

B. 客戶資產保管
   □ 客戶資產與公司資產隔離機制
   □ 冷錢包/熱錢包管理政策
   □ 私鑰管理程序（多簽要求）

C. 技術安全
   □ 智能合約安全審計報告（如有）
   □ 滲透測試報告
   □ 業務連續性計畫（BCP）
```

### Travel Rule 合規要求

VASP 須遵守 FATF Travel Rule（台灣已納入 AML 法規體系）：

```
觸發條件：單筆虛擬資產轉帳 ≥ USD 1,000（或等值）

發送方 VASP 須傳遞：
├── 發起人：姓名、帳號、地址（或國籍ID或出生日期+地點）
└── 受益人：姓名、帳號

合規方式（選一）：
├── 加入 TRUST 或 Sygna Bridge 等 Travel Rule 網路
├── 使用 OpenVASP 協議
└── 雙邊協議（適用大型機構間）

⚠️ 對方若非登錄 VASP（Unhosted Wallet）：
   須執行強化盡職調查（EDD）
```

## 監理沙盒申請（金融科技創新實驗）

### 適用時機

```
沙盒不是捷徑，以下情況才適合申請：

✓ 商業模式確實創新，現行法規無對應框架
✓ 需要在小規模測試中驗證合規可行性
✓ 有明確的後續取照路徑規劃

✗ 不適合：已有對應執照類型但不想申請
✗ 不適合：用於降低合規成本
✗ 不適合：境外業者繞過台灣執照要求
```

### 沙盒申請書核心要素

```
1. 創新說明
   - 現行法規如何構成障礙（具體條文引用）
   - 與現有服務的本質差異

2. 實驗計畫
   - 測試範圍：用戶數上限、交易量上限、地理範圍
   - 測試期間：最長 18 個月（可申請延長至 36 個月）
   - 成功指標定義

3. 消費者保護
   - 損失補償機制（如實驗失敗）
   - 資訊揭露規範
   - 退場機制

4. 退場計畫（Exit Plan）
   必須二擇一：
   ├── 取得完整執照並正式商業化
   └── 有序終止服務（含客戶通知、資金返還程序）
```

### 沙盒核准後仍適用的義務

```
❌ 沙盒不豁免的項目：
   - AML/CFT 義務（洗錢防制法仍完全適用）
   - 個人資料保護法
   - 消費者保護法
   - 刑事詐欺相關法規

✅ 沙盒豁免的項目：
   - 執照/登錄要求（實驗期間內）
   - 最低資本額要求（由 FSC 個案認定）
   - 特定業務限制（依核准範圍）
```

## 資本額規劃：實務計算

### 電子支付機構資本額時程規劃範例

```
假設：A 公司計畫申請電子支付機構許可
目標資本額：NT$5 億（法定最低）

建議分期：
Year 0（籌設申請）：繳足 NT$1 億（展示誠意）
Year 1（正式申請前）：增資至 NT$3 億
Year 1 Q4（正式申請）：增資至 NT$5 億（取具銀行查核報告）

股東出資能力文件：
├── 個人股東：近 3 年綜合所得稅申報書 + 存款/投資證明
├── 法人股東：近 3 年經會計師查核財務報表
└── 境外股東：母公司財報（須附中文翻譯）
```

### 負債比率要求（持續監理）

```
取照後，電子支付機構須維持：
├── 自有資本 ÷ 風險性資產 ≥ 10%（類銀行資本適足率概念）
└── 備付款項（客戶儲值餘額）必須 100% 保全（信託或保全）

備付款項保全計算：
保全金額 = Σ（所有用戶當前儲值餘額）
每日計算，須即時補足缺口
```

## 申請時間軸總覽

```
電子支付機構許可（樂觀情境）：

Month  1-3：Pre-filing 諮詢，確認業務範圍
Month  4-9：籌設申請書準備（業務計畫書為核心）
Month 10：送件籌設申請
Month 13-16：FSC 審查籌設申請（含補件）
Month 17：取得籌設核准
Month 18-21：完成公司登記、資本額繳足、系統建置
Month 22：送件正式許可申請
Month 25-34：FSC 審查正式申請（含實地查核）
Month 35：取得正式許可 → 開業

⚠️ 實際時程常因補件次數延長。
   業界平均：籌設到開業約 2-3 年。
```

## 常見退件原因彙整

| 退件原因 | 發生頻率 | 解決方式 |
|---------|---------|---------|
| 業務計畫書資金流向不清 | 高 | 繪製詳細資金流向圖，標示清算時點 |
| 備付款項保全機制不足 | 高 | 預先取得信託業者合作意向書 |
| AML 程序書過於籠統 | 高 | 參照 FSC 公告範本，加入具體閾值 |
| 董事/負責人不符適格性 | 中 | 提前確認，有犯罪紀錄者需法律意見 |
| 資本額來源無法說明 | 中 | 境外資金需附合法匯入證明 |
| 資訊安全規劃不足 | 中 | 委外滲透測試，提前取得報告 |
| 財務預測與業務規模不符 | 低 | 保守預測，附假設說明 |
