# 溫室氣體盤查實務：GHG Protocol 與 ISO 14064

## 主要標準對照

| 標準 | 涵蓋層級 | 主要用途 | 台灣採用度 |
|---|---|---|---|
| GHG Protocol Corporate Standard | 組織層 | 自願盤查、客戶回應 | 最廣 |
| GHG Protocol Scope 3 Standard | 組織層範疇 3 | 範疇 3 專用 | 普遍 |
| GHG Protocol Product Standard | 產品層 | 產品碳足跡 | 部分採用 |
| ISO 14064-1 | 組織層 | 可被驗證、法規採用 | 環境部指定 |
| ISO 14064-2 | 專案層 | 減量專案 | 碳交易使用 |
| ISO 14064-3 | 查證 | 第三方驗證 | 配合 14064-1 |
| ISO 14067 | 產品層 | 產品碳足跡 | 採用中 |

**台灣企業典型實務**：組織邊界依 GHG Protocol、內容對齊 ISO 14064-1，以支援第三方查證。

## GHG Protocol 七大管制氣體

| 氣體 | GWP（AR6 / 100 年） | 主要來源 |
|---|---|---|
| CO2（二氧化碳） | 1 | 燃燒、製程 |
| CH4（甲烷） | 29.8 | 廢棄物、畜牧、能源逸散 |
| N2O（氧化亞氮） | 273 | 農業、燃燒 |
| HFCs（氫氟碳化物） | 1,500–14,800 | 冷媒、泡沫劑 |
| PFCs（全氟碳化物） | 7,380–12,400 | 半導體、鋁業 |
| SF6（六氟化硫） | 25,200 | 電力設備 |
| NF3（三氟化氮） | 17,400 | 半導體 |

**注意**：GWP 係數每次 IPCC 評估報告更新。2026 年盤查多採 AR5 或 AR6 係數，視主管機關指定。

## 組織邊界（Organizational Boundaries）

### 三種方法

**營運控制法（Operational Control）**
- 納入可主導營運政策的實體
- **最常用**、最易界定
- 涵蓋所有子公司、部分合資

**財務控制法（Financial Control）**
- 納入財務控制的實體（通常 > 50% 持股）
- 與財報合併範圍一致
- 適合有大量少數股權投資者

**股權比例法（Equity Share）**
- 依持股比例計算
- 金融業、複雜股權結構用

**建議**：首次盤查採「營運控制法」、與財報合併一致。集團複雜者採財務控制法。

### 邊界內常見爭議
- **合資（JV）**：誰計？通常依主導營運者
- **委外製造（OEM / EMS）**：採購商通常不納入範疇 1+2，而計入範疇 3
- **加盟店**：依是否主導營運判定
- **辦公室租賃**：通常納入
- **員工遠距工作**：通常計入範疇 3 類別 7

## 範疇 1：直接排放

### 來源類別

**固定燃燒（Stationary Combustion）**
- 鍋爐、熔爐、加熱器
- 燃料：天然氣、重油、柴油、煤、生質
- 係數：IPCC 2006、AR5/AR6、台灣環境部

**移動燃燒（Mobile Combustion）**
- 公司車輛、堆高機、發電機
- 主要燃料：汽油、柴油、LPG
- 每公升燃料 × 排放係數

**製程排放（Process Emissions）**
- 化學反應直接排放（水泥、石化、鋁業）
- 電子業 PFCs、SF6

**逸散排放（Fugitive Emissions）**
- 冷媒（HFCs）
- 電力設備 SF6
- 管線天然氣洩漏
- 滅火劑

### 計算範例（範疇 1）

```
天然氣鍋爐：
月消耗 100,000 Nm³
係數：2.10 kg CO2e / Nm³
月排放：100,000 × 2.10 = 210,000 kg = 210 tCO2e
```

## 範疇 2：外購能源間接排放

### 兩種計算方法

**位置基礎（Location-based）**
- 用電網平均排放係數
- 台灣 2024 電網係數：約 0.494 kg CO2e / 度（每年更新）
- **必須揭露**

**市場基礎（Market-based）**
- 考慮綠電採購、T-REC、RE100 承諾
- 有簽約綠電：該部分採 0 tCO2e（具憑證依據）
- 無特殊合約：用「殘餘排放係數」（Residual Mix）
- **建議同時揭露**

### 外購蒸汽與熱
- 罕見但需計入
- 來自集中能源中心或熱電聯產

### 計算範例（範疇 2）

```
外購電力：
月用電 5,000,000 度
位置基礎係數：0.494 kg / 度
月排放（位置）：5,000,000 × 0.494 = 2,470,000 kg = 2,470 tCO2e

若其中 2,000,000 度為綠電 PPA：
市場基礎月排放：(5,000,000 - 2,000,000) × 0.494 = 1,482 tCO2e
```

## 範疇 3：十五大類

### 類別 1：採購商品與服務

**計算方法**：
- **支出基礎法**（Spend-based）：採購金額 × 係數（如 EEIO 係數表）
- **平均數據法**（Average Data）：採購量 × 每單位平均碳排
- **混合法**（Hybrid）：實際數據 + 支出補充
- **實際數據法**（Supplier-specific）：供應商實測 → 準確但難取得

**建議**：首次盤查用支出基礎（快速）、重點供應商採實際數據。

### 類別 2：資本財
設備、建築購置時的嵌入碳。攤提 or 當期認列有爭議。

### 類別 3：燃料與能源相關活動
- 上游採礦 / 輸配 / 蒸餾
- 計算：燃料活動 × 上游係數

### 類別 4：上游運輸
- 採購物品的運輸
- 計算：運輸距離 × 重量 × 運輸方式係數

### 類別 5：廢棄物處理
- 垃圾焚化、掩埋、回收
- 係數依處理方式差異大

### 類別 6：商務差旅
- 飛行（短中長程）、高鐵、汽車、住宿
- 多工具 / 平台可自動計算

### 類別 7：員工通勤
- 可以問卷估計或使用平均值
- 遠距工作排放亦計入

### 類別 8：上游租賃資產
- 租賃設施 / 車輛的排放

### 類別 9：下游運輸
- 出貨給客戶的運輸

### 類別 10：銷售產品加工
- 若賣中間產品給客戶再加工

### 類別 11：銷售產品使用
- **消費性電子、汽車、家電通常最大**
- 需推估產品壽命 + 使用情境

### 類別 12：銷售產品報廢
- 產品壽命結束後的處理

### 類別 13：下游租賃資產
- 出租給他人使用的資產

### 類別 14：加盟
- 加盟店排放（若適用）

### 類別 15：投資
- 金融業最關鍵類別
- 使用 PCAF（Partnership for Carbon Accounting Financials）方法

## SBTi 對範疇 3 的要求

- 若範疇 3 占總排放 ≥ 40% → 必須設定範疇 3 目標
- 涵蓋率 ≥ 67%（一般企業）或 ≥ 90%（FI 金融機構）
- 目標可為絕對減量、強度減量、或供應商參與目標（Supplier Engagement Target）

## 係數來源與選用

### 常用係數資料庫

**國際**
- **IPCC 排放係數資料庫**（EFDB）
- **UK DEFRA / BEIS**：每年更新、涵蓋商務差旅、運輸
- **US EPA**：美國電網、燃料
- **Ecoinvent**：產品生命週期資料庫（需付費）
- **GHG Protocol 範疇 3 計算工具**

**台灣**
- **環境部國家溫室氣體排放係數資料庫**（最權威）
- **台電電網排放係數**（每年 6 月公布前一年度）
- **中油燃料熱值 / 係數**

### 選用原則
- 優先使用台灣本土係數（範疇 1、2）
- 若無本土係數採國際標準
- 記錄版本年份，每次更新保持一致
- 基準年與當年度採同版本係數（否則需重編）

## 盤查資料管理

### 典型資料類型

**能源類**
- 電費單（每月）
- 天然氣 / 燃料發票
- 車輛加油紀錄

**差旅類**
- 出差申請系統
- 住宿訂房資料

**採購類**
- ERP 採購金額彙總
- 重點供應商原物料明細

**廢棄物**
- 廢棄物處理委外合約 + 月報

### 資料品質分級
- **L1 實測**：儀表、發票、供應商實測
- **L2 推算**：活動量 × 標準係數
- **L3 估計**：支出基礎、行業平均
- **L4 粗估**：類比、假設

**建議**：範疇 1+2 以 L1-L2 為主、範疇 3 以 L2-L3 為主、重點類別逐年升級至 L1-L2。

## 首次盤查實務 18 個月時程

| 月份 | 主要工作 |
|---|---|
| 1–2 | 組織邊界界定、專案組成 |
| 3–4 | 資料類別與蒐集流程設計 |
| 5–6 | 範疇 1+2 資料蒐集 |
| 7–8 | 範疇 1+2 計算、文件化 |
| 9–10 | 範疇 3 熱點識別、主要類別計算 |
| 11–12 | 彙整與不確定性分析 |
| 13–14 | 內部審閱 / 修訂 |
| 15–16 | 第三方查證（若執行） |
| 17–18 | 對外揭露、客戶回應 |

## 常見盤查錯誤與修正

### 錯誤 1：雙重計算
- 範疇 2 電力 + 範疇 3 類別 3 上游電力的重複
- 自用綠電 + T-REC 的重複
- 修正：清楚界定各範疇邊界、不交叉

### 錯誤 2：活動量單位錯誤
- 電費元 → 直接乘係數（錯）應該先換算成度數
- 修正：單位一致性檢查

### 錯誤 3：範疇 3 類別跳過
- 類別 11（產品使用）省略 → 實際可能占大部分
- 修正：列出所有 15 類、顯著性審查、公開未納入理由

### 錯誤 4：基準年不當
- 選單一年特殊事件多 → 目標設計扭曲
- 修正：採「3 年平均」或公開單年的特殊因素

### 錯誤 5：邊界變動未重編
- 併購、分割、出售 → 基準年需調整
- 修正：依 GHG Protocol 重編規則更新

## 查證（Verification）

### ISO 14064-3
- 第三方查證機構
- 兩級：**合理保證**（高信心）/ **有限保證**（中信心）
- 產出：查證聲明書

### 台灣本土查證機構
- 工研院、金屬中心、SGS、BV、TÜV 等

### 查證流程
1. 合約與範圍界定
2. 文件審查
3. 現場查證（抽樣）
4. 查證報告
5. 主要發現溝通

### 查證的用途
- 符合法規（如台灣年排 > 2.5 萬噸需強制查證）
- 客戶信任（CDP、Apple 供應商）
- SBTi 相關要求
